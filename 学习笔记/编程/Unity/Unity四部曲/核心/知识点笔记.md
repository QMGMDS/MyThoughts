## 图片导入相关设置
### 图片导入概述

- Unity 支持的相关图片素材：
	1. BMP：是 Windows 操作系统的标准图像文件格式，特点是几乎不进行压缩，占磁盘空间大
	2. TIF：基本不损失图片信息的图片格式，缺点是体积大
	3. JPG：一般指 JPEG 格式，属于有损压缩格式，能够让图像压缩在很小的存储空间，一定程度上会损失图片数据，无透明通道
	4. PNG：无损压缩算法的位图格式，压缩比高，生成文件小，有透明通道
	5. TGA：支持压缩，使用不失真的压缩算法，还支持编码压缩。体积小，效果清晰，兼备BMP的图像质量和JPG的体积优势，有透明通道
	6. PSD：是PhotoShop（PS）图形处理软件专用的格式，通过一些第三方工具或自制工具可以直接将PSD界面转为UI界面
	7. EXR
	8. GIF
	9. HDR
	10. IFF
	11. PICT
	12. 等等

### 参数设置-高级设置

![[Pasted image 20260411220318.png]]
- 什么是 `MipMaps` ？
	在三维计算机图形的贴图渲染中有一个常用的技术被称为 Mipmapping
	该技术指为了加快渲染速度和减少图像锯齿，贴图被处理成由一系列被预先计算和优化过的图片组成的文件，这样的贴图文件被称为 `MipMap`
	`MipMap` 需要占用一定的内存空间
	
	`Mipmap` 中每一个层级的小图都是主图的一个特定比例的缩小细节的复制品
	虽然在某些必要的视角，主图仍然会被使用，来渲染完整的细节
	但是当贴图被缩小或者只需要从远距离观看时，`Mipmap` 就会转换到适当的层级
	
	因为 `Mipmap` 贴图需要被读取的像素远少于普通贴图，所以渲染的速度得到了提升，并且操作的时间减少了，因为 `Mipmap` 的图片已经是做过抗锯齿处理的，从而减少了实时渲染的负担，放大和缩小也因为 `Mipmap` 而变得更有效率
	
	如果贴图的基本尺寸是256x256像素的话,它 `Mipmap` 就会有8个层级
	每个层级是上一层级的四分之一的大小
	依次层级大小就是：128x128 64x64 32x32 16x16 8x8 4x4 2x2 1x1(一个像素)
	
	说人话：开启MipMap功能后，Unity会帮助我们根据图片信息生成n张不同分辨率的图片
	在场景中会根据我们离该模型的距离选择合适尺寸的图片用于渲染，提升渲染效率，以达到用内存换性能的目的

## Sprite
### Sprite Editor

精灵图片编辑器
非 2D 项目需要安装 2D Sprite 包才能使用 SpriteEditor
![[Pasted image 20260411224714.png]]

通过左上角切换不同编辑模式：
1. Sprite Editor
基础图片设置（右下角窗口）
主要用于设置单张图片的基础属性

2. Custom Outline（决定渲染区域）  
自定义边缘线设置，可以自定义精灵网格的轮廓形状
默认情况下不修改都是在矩形网格上渲染,边缘外部透明区域会被渲染，浪费性能
使用自定义轮廓，可以调小透明区域，提高性能

3. Custom Physics Shape（决定碰撞判断区域）
自定义精灵图片的物理形状，主要用于设置需要物理碰撞判断的2D图形
它决定了之后产生碰撞检测的区域

4. Secondary Textures(为图片添加特殊效果)
次要纹理设置，可以将其它纹理和该精灵图片关联
着色器可以得到这些辅助纹理然后用于做一些效果处理
让精灵应用其它效果

### Sprite Renderer

精灵渲染器，有了它图片素材才在场景中有了显示
![[Pasted image 20260412084457.png]]
- `Sprite`：渲染的精灵图片
- `Color`：定义着色，一般没有特殊需求不会修改
- `Filp`：水平或竖直翻转精灵图片
- `Draw Mode`：绘制模式，当尺寸变化时的缩放方式
- `Mask Interaction`：与精灵遮罩交互时的方式
- `Sprite Sort Point`：计算摄像机和精灵之间距离时，使用精灵中心Center还是轴心点Pivot，一般情况下不用修改
- `Material`：材质，可以使用一些自定义材质来显示一些特殊效果一般情况不修改。默认材质是不会受到光照影响的，如果想要受光照影响，可以选择Default-Diffuse
- `Additional Settings`：高级设置
	`Sorting Layer`：排序层选择
	`Order in Layer`：层级序列号，数值越大约会显示在前面

### Sprite Mask

![[Pasted image 20260412085639.png]]
- `Sprite`：遮罩图片
- `Alpha Cutoff`：如果Alpha包含透明区域和不透明区域之间的混合（半透明），则可以手动确定所显示区域的分界点（0~1）
- `Custom Range`：自定义遮罩范围，开启后可以设置遮罩的范围，按照排序层来划分
- `Sprite Sort Point`：计算摄像机和精灵之间距离时，使用精灵中心Center还是轴心点Pivot，一般情况下不用修改

### Sorting Group

![[Pasted image 20260412090222.png]]
`SortingGroup` 是排序分组的意思
它的主要作用就是对多个精灵图片进行分组排序
主要作用是对于需要分层的2D游戏用于整体排序

### Sprite Atlas

精灵图集，也就是打图集
**打图集的目的就是为了减少 DrawCall ，提高性能**
Unity 准备了打图集的功能，在 `Edit/Project Setting/Editor` 中进行启用
![[Pasted image 20260412121749.png]]
Sprite Packer 即精灵包装器，选择是否开启此功能，这就是 Unity 自带的图集工具

- `Disabled`：默认设置，不会打包图集
- Legacy Sprite Packer：传统打包模式，多了一个设置图片之间的间隔距离参数
	- `Enabled For Builds(Legacy Sprite Packer)`：Unity仅在构建时打包图集，在编辑模式下不会打包图集
	- `Always Enabled(Legacy Sprite Packer)`：Unity在构建时打包图集，在编辑模式下运行前会打包图集
	
	`Padding Power`：选择打包算法在计算打包的精灵之间以及精灵与生成的图集边缘之间的间隔距离，这里的数字代表2的n次方
- `Enabled For Build`：Unity进在构建时打包图集，在编辑器模式下不会打包
- `Always Enabled`：Unity在构建时打包图集，在编辑模式下运行前会打包图集

启用了 Sprite Packer 就可以在 Project 窗口中右键 Create/Sprite Atlas 创建图集

恒定力是一个特殊的脚本

# 2D 动画
## 序列帧动画

- 概述
	以固定时间间隔按序列切换图片，就是序列帧动画的本质
	当固定时间间隔足够短时，肉眼就会认为图片是连续动态的，进而形成动画
	即动画就是会动的画面，序列帧动画就是连续播放的图片

## 2D 骨骼动画

- 概述
	2D 骨骼动画是利用 3D 骨骼动画的制作原理，以节约美术资源为目的，依靠算法和性能去节约内存的动画

- 如何制作 2D 骨骼动画？
	共有两种主流方式：
	1. 使用 Unity2018 新加功能 2D Animation 工具制作
	2. 使用跨平台骨骼动画制作工具 Spine 制作
	下面就是介绍两种工具如何去制作 2D 骨骼动画

### 2D Animation 制作动画

使用 2D Animation 可以为单张图片和图集资源进行骨骼动画的制作

- 配置
	需要在 Package Manager 窗口中搜索 2D Animation 并安装
	导入工具后在 Sprite Editor 窗口会多一个选项 Skinning Editor
	该选项的窗口即为专门制作 2D 骨骼动画的工作区

- 制作 2D 骨骼动画的流程
	绑骨骼 -> 蒙皮 -> 设权重 -> 添加关键帧
	骨骼：带动运动的 “ 操控器 ”
	蒙皮：决定 “ 皮肤 ” 如何跟随骨骼运动的 “ 连接规则 ”
	权重：连接规则里最重要的一个参数，用来精确分配每块骨骼对皮肤上某个点的影响程度

- 拓展
	- 什么是 PSB/PSD 文件？
		PSB 和 PSD 两种格式，都是 PS(Photoshop) 这款软件用于保存图像处理数据的文件格式
		PSD和PSB两种格式并没有太大的区别
		最大的区别是 PSD 格式兼容除 PS 以外的其它一些软件
		而 PSB 只能用 PS 打开
		在 Unity 中制作骨骼动画时，官方建议使用 PSD 格式的美术资源
	
	- 什么是反向动力学 (IK)？
		在骨骼动画中，子关联父的构建骨骼的方法被称为正向动力学
		它的表现形式是，子骨骼（关节）的位置根据父骨骼（关节）的旋转而改变
		反向动力学和正向动力学恰恰相反
		它的表现形式是，子骨骼（关节）末端的位置改变会带动自己以及自己的父骨骼（关节）旋转
