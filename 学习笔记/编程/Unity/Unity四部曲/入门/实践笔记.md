## 场景切换

- 注意：
	对于要切换的目标场景
	需要将其先加载到场景列表中
	否则场景的切换无法实现
	
	如何将目标场景加载到场景列表中？
	Unity 编辑器工具栏中 `File/Build Settings` 打开导出窗口
	像这样：![[Pasted image 20260317134544.png]]
	上方的 `Scenes In Build` 即场景列表
	点击 `Add Open Scenes` 即可将现在 Hierachy 窗口中的所有场景加载到列表中
	也可以将场景文件拖拽到列表中直接添加
	场景列表中每个场景中的序号代表该场景在游戏运行时的加载顺序
	序号越小的场景越先被加载
	0 序号表示该场景在游戏运行时即被加载 一般作为游戏主界面或登录界面

## 模型资源导入

- 模型由什么构成？
	骨 (骨骼)：非必须 有动作的模型才需要
	肉 (网格面片)：必须 决定了模型的轮廓
	肉 (网格面片)：必须 决定了模型的颜色效果

- Unity支持的模型格式
	.fbx
	.dae
	.3ds
	.dxf
	.obj

- 关于Unity模型导出的官方文档
	https://docs.unity.cn/cn/2019.4/Manual/CreatingDCCAssets.html

- 导出的模型的重要规则
	Unity中的模型面朝向朝模型坐标轴的 Z 轴
	缩放大小单位要注意

# 数据持久化 PlayerPrefs


什么是数据持久化？
数据持久化就是将游戏运行时产生的数据存储到硬盘当中
即将内存的数据存储到硬盘当中进行保存
在下一次游戏运行时通过读取硬盘中的数据实现上一次游戏数据的继承
在这一过程中数据被持久保存在硬盘里
这就是数据持久化 描述数据的存取读取的两个过程
而 PlayerPrefs 就是实现数据持久化的方法之一

## 小项目

**目的：创建一个 PlayerPrefsDataMgr 的单例类**
- 只需要调用 `SaveData(object data, string keyName)` 的方法 传入任意数据实例与自定义名称便可以自动使用 PlayerPrefs 的方法将数据存储起来
- 只需要调用 `LoadData(Type type, string keyName)` 的方法 传入想要获取的对象类型和先前的自定义名称便可以自动获取到通过 PlayerPrefs 方法存储起来的数据

# GUI

## GUI 工作原理和主要作用

- 概述
	GUI 即由代码驱动的 UI 系统
	它的特点是只能在游戏运行时才能看到 UI 显示
	它一般作为程序员的调试工具 而不是玩家 UI 显示
	可以为脚本组件创建自定义检视面板
	或创建新的编辑器窗口和工具以拓展 Unity 本身
