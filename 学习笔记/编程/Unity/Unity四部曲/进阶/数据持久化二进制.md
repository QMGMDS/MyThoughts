# 概述
采用二进制的方式进行数据持久化就是将数据转换成字节数据，即一连串的 0 和 1，将这些字节数据以文件的形式存储起来，以及读取这些二进制数据文件反过来转换成变量数据
这么做主要有两个好处：
1. 节约存储空间
2. 提升安全性
在网络通信中直接传输的数据也是字节数据（2进制数据）

# 变量数据类型与字节数组的转换
实现转换的重要类就是 `BitConverter` ，学习转换就是学习相关的 API

- 非 string 类型变量的转换
```C#
// 1.将各类型转字节
// 调用 GetBytes() 方法 该方法有十个重载 可以将以下变量类型转换成字节数组
// uint ushort ulong int short long float double bool char
// 一个字节有8位 可以表示 0~255
byte[] bytes = BitConverter.GetBytes(256);
// 字节数组的内容（顺着读） 0 1
// 有点像倒着存储的感觉 所以倒着读就是
// 表示 0000 0001 0000 0000 对应数字 256

// 2.字节数组转各类型
// 调用 To 的方法就可以将字节数组转成对应的数据
// 参数一：要转换的字节数组
// 参数二：从哪里开始读取字节，根据要转换的目标数据来依次往后读取对应字节数
int i = BitConverter.ToInt32(bytes, 0);
```

- string 类型变量的转换
由于文字和数字等语言符号在不同的编码格式下对应的编码不同
编码之间的差异性会导致数据的错误，这就是 string 类型变量转二进制时需要注意的地方
解决思路就是利用 C# 提供的编码格式类 `Encoding` ，让编码格式统一
```C#
// 1.将字符串以指定编码格式转字节
byte[] bytes = Encoding.UTF8.GetBytes("异环");

// 2.字节数组以指定编码格式转字符串
string s = Encoding.UTF8.GetString(bytes);

// 除了 UTF8 格式之外
// Encoding 还有很多其他格式 但 UTF8 是游戏开发中最常用的编码格式
```

# 文件
在电脑上可以在操作系统中创建删除修改文件，增删查改各种各样的文件类型
下面介绍的就是通过代码来实现这些操作的 API

## 文件相关操作
学习代码操作文件就是学习 `File` 类中提供的 API
下面的相关操作基本都是直接操作整个文件的内容
如果要精确的操作文件的具体内容需要用到文件流相关操作

- 判断一个文件是否存在
```C#
// 判断 Application.dataPath 该路径下是否有 原神.nb 这个文件
// 参数：填入需要判断的文件具体路径
if (File.Exists(Application.dataPath + "/原神.nb"))
    print("文件存在");
else
    print("文件不存在");
```

- 创建文件
```C#
// 将 原神.nb 该文件创建出来
// 返回文件流对象 FileStream
File.Create(Application.dataPath + "/原神.nb");
```

- 写入文件
如下的写入方法都是直接覆盖文件整体的所有内容
并且要写入文件的前提是该文件不被其他进程所占用
```C#
// 将指定 字节数组 写入到指定路径的文件中
byte[] bytes = BitConverter.GetBytes(999);
File.WriteAllBytes(Application.dataPath + "/原神.nb", bytes);

// 将指定 string数组 一行行写入到指定路径的文件中
string[] strs = new string[] { "123", "321", "鸣潮牛逼" };
File.WriteAllLines(Application.dataPath + "/原神.nb", strs);

// 将指定 string 写入到指定路径的文件中
File.WriteAllText(Application.dataPath + "/原神.nb", "终末地才是最牛逼的");
```

- 读取文件
```C#
// 读取字节数据
byte[] bytes = File.ReadAllBytes(Application.dataPath + "/原神.nb");

// 读取所有行信息
string[] strs = File.ReadAllLines(Application.dataPath + "/原神.nb");

// 直接读取所有文本信息
print(File.ReadAllText(Application.dataPath + "/原神.nb"));
```

- 删除文件
删除文件的前提是该文件不被其他进程所占用
```C#
File.Delete(Application.dataPath + "/原神.nb");
```

- 复制文件
```C#
// 参数一：要复制的现有文件
// 参数二：复制出来的目标文件
// 参数三：如果目标文件已经存在是否选择覆盖 默认为false
// 如果目标文件已经存在且不覆盖会直接报错
File.Copy(Application.dataPath + "/原神.nb", Application.dataPath + "/鸣潮.nb", true);
```

- 文件替换
```C#
// 参数一：替换的内容文件路径
// 参数二：被替换的目标文件
// 参数三：被替换的文件的备份路径
File.Replace(Application.dataPath + "/鸣潮.nb", Application.dataPath + "/原神.nb", Application.dataPath + "/原神备份.nb");
```

## 文件流相关操作
文件流的主要作用是用于读写文件的细节，以读写字节的形式处理文件
通过文件流一部分一部分的读写数据
比如先存一个int（4个字节），再存一个string（n个字节）
利用 `FileStream` 可以以流式逐个读写
学习文件流的相关操作就是学习 C# 提供的 `FileStream` 文件流类

### 通过文件流打开或创建指定文件

- 方式一：通过构造函数
```C#
// 参数一：打开的文件路径
// 参数二：打开模式：
//      CreateNew:创建新文件 如果文件存在 则报错
//      Create:创建文件，如果文件存在 则覆盖
//      Open:打开文件，如果文件不存在 报错
//      OpenOrCreate:打开或者创建文件根据实际情况操作
//      Append:若存在文件，则打开并查找文件尾，或者创建一个新文件
//      Truncate:打开并清空文件内容
// 参数三：访问模式
// 参数四：共享权限：
//      None 谢绝共享
//      Read 允许别的程序读取当前文件
//      Write 允许别的程序写入该文件
//      ReadWrite 允许别的程序读写该文件
FileStream fs = new FileStream(Application.dataPath + "/原神.nb", FileMode.Create, FileAccess.ReadWrite, FileShare.None);
```

- 方式二：通过 `File.Create` 的成员方法
```C#
// 参数一：打开的文件路径
// 参数二：缓存大小
// 参数三：描述如何创建或覆盖该文件（不常用）
//      Asynchronous 可用于异步读写
//      DeleteOnClose 不在使用时，自动删除
//      Encrypted 加密
//      None 不应用其它选项
//      RandomAccess 随机访问文件
//      SequentialScan 从头到尾顺序访问文件
//      WriteThrough 通过中间缓存直接写入磁盘
FileStream fs2 = File.Create(Application.dataPath + "/原神.nb");
```

- 方式三：通过 `File.Open` 的成员方法
```C#
// 参数一：路径
// 参数二：打开模式：
//      CreateNew:创建新文件 如果文件存在 则报错
//      Create:创建文件，如果文件存在 则覆盖
//      Open:打开文件，如果文件不存在 报错
//      OpenOrCreate:打开或者创建文件根据实际情况操作
//      Append:若存在文件，则打开并查找文件尾，或者创建一个新文件
//      Truncate:打开并清空文件内容
FileStream fs3 = File.Open(Application.dataPath + "/原神.nb", FileMode.Open);
```

### 文件流的重要属性和方法

```C#
FileStream fs = File.Open(Application.dataPath + "Lesson3.tang", FileMode.OpenOrCreate);

// 文件文本字节长度
print(fs.Length);

// 判断文件是否可写
if (fs.CanRead) { }

// 判断文件是否可读
if (fs.CanWrite) { }

// 强制将字节写入文件 防止程序异常退出时丢失数据
fs.Flush();

// 关闭流 释放文件句柄 会自动调用 Flush()
fs.Close();

// 缓存资源销毁回收 会自动调用 Close()
fs.Dispose();
```

### 通过文件流写入

- 写入字节
```C#
FileStream fs = new FileStream(Application.persistentDataPath + "/原神.nb", FileMode.OpenOrCreate, FileAccess.Write);
byte[] bytes = BitConverter.GetBytes(999);

// 写入字节
// 参数一：写入的字节数组
// 参数二：数组中的开始索引 写入的开始
// 参数三：写入多少个字节
fs.Write(bytes, 0, bytes.Length);
```

- 写入字符串
```C#
FileStream fs = new FileStream(Application.persistentDataPath + "/原神.nb", FileMode.OpenOrCreate, FileAccess.Write);
byte[] bytes = Encoding.UTF8.GetBytes("异环牛逼");

// 写入字符串
// 先写入长度 长度转字节写入 长度int为4字节
fs.Write(BitConverter.GetBytes(bytes.Length), 0, 4);
// 再写入字符串具体内容
fs.Write(bytes, 0, bytes.Length);
```

### 通过文件流读取
假设已经有 `FileStream fs = File.Open(Application.persistentDataPath + "/原神.nb", FileMode.Open, FileAccess.Read);` 文件有内容为：
`111鸣朝牛逼` 转成的 二进制数据

- 读取第一个整形
```C#
byte[] bytes = new byte[4];

// 参数一：用于存储读取到的字节数组的容器
// 参数二：容器中开始的位置
// 参数三：读取多少个字节装入容器
// 返回值：当前流索引前进了几个位置
int index = fs.Read(bytes, 0, 4);
int i = BitConverter.ToInt32(bytes, 0);
```

- 读取第二个字符串
```C#
 // 装载字符串长度
byte[] bytes = new byte[4];

// 先读长度
index = fs.Read(bytes, 0, 4);
int length = BitConverter.ToInt32(bytes, 0);

// 再读字符串
// 根据存储的字符串字节数组的长度来声明一个新的字节数组
// 装载读取出来的字符串
bytes2 = new byte[length];
index = fs.Read(bytes2, 0, length);
string s = Encoding.UTF8.GetString(bytes2);
```

### 更安全的使用文件流对象

```C#
// using 关键字重要用法
// using (申明一个引用对象)
// {
//      使用对象
// }
// 无论发生什么情况 当 using 语句块结束后 
// 会自动调用该引用对象的销毁方法 避免忘记销毁或关闭流
// using 是一种更安全的使用方法

// 示例
using (FileStream fs3 = File.Open(Application.persistentDataPath + "/原神.nb", FileMode.Open, FileAccess.Read))
{
    // 一开始就申明一个和文件字节数组长度一样的容器
    byte[] bytes3 = new byte[fs3.Length];
    fs3.Read(bytes3, 0, (int)fs3.Length);
    fs3.Dispose();
    // 读取整数
    print(BitConverter.ToInt32(bytes3, 0));
    // 得去字符串字节数组的长度
    int length2 = BitConverter.ToInt32(bytes3, 4);
    // 得到字符串
    print(Encoding.UTF8.GetString(bytes3, 8, length2));
}
```

## 文件夹相关操作
通过代码的形式增删查改文件夹
学习文件夹相关操作就是学习 `Directory` 类

- 判断文件夹是否存在（之前 `Flie` 中是判断文件是否存在）
```C#
if (Directory.Exists(Application.dataPath + "/二游"))
    print("存在文件夹");
else
    print("文件夹不存在");
```

- 创建文件夹
```C#
// DirectoryInfo 为目录信息类 可以通过它获取文件夹的更多信息
DirectoryInfo info = Directory.CreateDirectory(Application.dataPath + "/二游");
```

- 删除文件夹
```C#
// 参数一：删除文件夹的路径
// 参数二：是否删除非空目录
// true 将删除整个目录
// false 仅当该目录为空时才可删除 非空删除直接报错
Directory.Delete(Application.dataPath + "/二游");
```

- 查找文件夹和文件
```C#
// 得到指定路径下所有文件夹名
string[] strs1 = Directory.GetDirectories(Application.dataPath);
for (int i = 0; i < strs.Length; i++)
{
    print(strs1[i]);
}

// 得到指定路径下所有文件名
string[] strs2 = Directory.GetFiles(Application.dataPath);
for (int i = 0; i < strs.Length; i++)
{
    print(strs2[i]);
}
```

- 移动文件夹
```C#
// 参数一：被移动的文件夹
// 参数二：移动到的目标位置
// 移动会把文件夹中的所有内容一起移到新的路径
// 如果第二个参数所在的路径已经存在了一个文件夹 会报错
Directory.Move(Application.dataPath + "/二游", Application.dataPath + "/原神");
```

- DirectoryInfo 目录信息类相关
```C#
// DirectoryInfo 目录信息类
// 我们可以通过它获取文件夹的更多信息
// 它主要出现在两个地方
// 1.创建文件夹方法的返回值
DirectoryInfo dInfo = Directory.CreateDirectory(Application.dataPath + "/数据持久化123");
// 全路径
print(dInfo.FullName);
// 文件名
print(dInfo.Name);

// 2.查找上级文件夹信息
dInfo = Directory.GetParent(Application.dataPath + "/数据持久化123");
// 全路径
print(dInfo.FullName);
// 文件名
print(dInfo.Name);

// 重要方法
// 得到所有子文件夹的目录信息
DirectoryInfo[] dInfos = dInfo.GetDirectories();

// FileInfo文件信息类
// 我们可以通过DirectoryInfo得到该文件下的所有文件信息
FileInfo[] fInfos = dInfo.GetFiles();
for (int i = 0; i < fInfos.Length; i++)
{
    print("**************");
    print(fInfos[i].Name); // 文件名
    print(fInfos[i].FullName); // 路径
    print(fInfos[i].Length); // 字节长度
    print(fInfos[i].Extension); // 后缀名
}
```
