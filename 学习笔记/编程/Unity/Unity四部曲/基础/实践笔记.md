# 数据持久化 XML

## XML 基本语法
元素和属性分别为记录数据信息的两种方式
属性也可以用作记录额外信息
注意：
1. 元素标签必须配对
2. 属性必须有引号
3. 命名规则和 C# 变量名命名规则一致
检查 XML 语法是否出错网址：
https://www.runoob.com/xml/xml-validator.html

- 示例
```xml
<?xml version="1.0" encoding="UTF-8"?>
<book>
    <title>XML入门教程</title>
    <author>张三</author>
    <price currency="CNY">49.00</price>
    <year>2023</year>
    <publisher>人民邮电出版社</publisher>
</book>
```

- 注释
`<!---->` 中写的内容为注释
```xml
<!--这里填写注释内容-->
<!--
这里填写注释内容
-->
```

- 固定内容
必须填写且必须在 XML 文件的第一行
代表 XML 的版本和编码格式
```xml
<?xml version="1.0" encoding="UTF-8"?>
```

- 元素
```xml
<!-- 一定要有一个根节点 -->
<PlayerInfo>
	<name>小明</name>
	<atk>100</atk>
	<ItemList>
		<Item>
			<id>1</id>
			<num>10</num>
		</Item>
		<Item>
			<id>2</id>
			<num>20</num>
		</Item>
	</ItemList>
</PlayerInfo>
```

- 属性
```xml
<!--属性就是在元素标签后面空格添加的内容-->
<!--注意：属性必须用引号包裹 单引号双引号都可以-->
<Friend name ="小明" age='8'>我的朋友</Friend>

<!--如果使用属性记录信息而不像使用元素记录 简写如下-->
<Friend name ="小明" age='8'/>
```

```xml
<PlayerInfo>
	<name>小明</name>
	<atk>100</atk>
	<ItemList>
		<Item id="1" num="10"/>
		<Item id="2" num="10"/>

```

## C\# 读取存储 XML
### XML 文件的存放位置
存放位置与文件夹的特点息息相关，详见 [[学习笔记/编程/Unity/Unity四部曲/基础/知识点笔记#特殊文件夹|特殊文件夹]]
1. 只读不写的 XML 文件可以放在 Resources 或者 StreamingAssets 文件夹下
2. 动态存储的 XML 文件放在 persistentDataPath 文件夹下

### XML 文件的读取
- 概述
读取 XML 文件的方式是：
1. 先加载 XML 文件
2. 调用方法查找父节点，进而通过父节点查找子节点
3. 通过访问节点 `InnerText` 属性进而获取到存储的数据

因此有两个重要类：
1. `XmlDocument` 专门存储 XML 文件信息类
2. `XmlNode` 代表每一个节点对象


- 拓展
C# 读取 XML 的方式有三种：
1. XmlDocument（把数据加载到内存中，方便读取）
2. XmlTextReader（以流形式加载，内存占用更少，但是是单向只读，使用不是特别方便，除非有特殊需求，否则不会使用）
3. Linq


- XML 文件的加载
```C#
// 1.调用 Resources 中加载资源的方法
// 将 XML 文件内容加载成文本
// 在 Resources 文件夹查找 TestXml 名称文件
TextAsset asset = Resources.Load<TextAsset>("TestXml");
// 通过类 XmlDocument
// 调用其中 LoadXml() 将字符串转为 XML 对象
XmlDocument xml = new XmlDocument();
xml.LoadXml(asset.text);

// 2.调用 XML 信息类中加载资源的方法
// 通过绝对路径来查找 XML 文件
XmlDocument xml = new XmlDocument();
xml.Load(Application.streamingAssetsPath + "/TestXml.xml");
```


- XML 文件节点的查找（以该XML文件为例）
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Root>
	<name>曹哲</name>
	<age>18</age>
	<Item id="1" num="10"/>
	<Friend>
		<name>小明</name>
		<age>8</age>
	</Friend>
	<Friend>
		<name>小红</name>
		<age>10</age>
	</Friend>
</Root>
```
```C#
XmlDocument xml = new XmlDocument();
xml.Load(Application.streamingAssetsPath + "/TestXml.xml");
// 通过节点名字去查找单个节点
// XmlNode 即节点信息类
XmlNode root = xml.SelectSingleNode("Root");
// 然后顺着上级节点查找下级节点
XmlNode nodeAge = root.SelectSingleNode("age");
XmlNode nodeItem = root.SelectSingleNode("Item");
// 通过 "/" 更快捷的找多层子节点
XmlNode nodeItem = xml.SelectSingleNode("Root/Item");

// 通过节点名字去查找多个同名节点
XmlNodeList friendList = root.SelectNodes("Friend");
```


- XML 文件节点的内容读取（以节点查找中内容为例）
```C#
// 获取节点包裹的元素信息
print(nodeName.InnerText); // 打印"曹哲"

// 获取属性包裹的元素信息
// 方式一：中括号获取信息
print(nodeItem.Attributes["id"].Value);
print(nodeItem.Attributes["num"].Value);
// 方式二：函数方法获取信息
print(nodeItem.Attributes.GetNamedItem("id").Value);  
print(nodeItem.Attributes.GetNamedItem("num").Value);

// 对于 SelectNodes 方法中找到的多个节点进行遍历
// 方式一：迭代器遍历
foreach (XmlNode item in friendList)
{
    print(item.SelectSingleNode("name").InnerText);
    print(item.SelectSingleNode("age").InnerText);
}
// 方式二：通过 for 循环遍历
// 通过 XmlNodeList 中的成员变量 Count 可以得到节点数量
for (int i = 0; i < friendList.Count; i++)
{
	print(friendList[i].SelectSingleNode("name").InnerText);
	print(friendList[i].SelectSingleNode("age").InnerText);
}
```

### XML 文件的存储
- 概述
读取 XML 文件的方式是：
1. 先创建一个 XML 文件信息类对象
2. 添加固定版本信息
3. 编写文件的 XML 格式节点信息
4. 调用 API 将文件保存到合适路径

因此有三个重要类：
1. `XmlDocument` 专门存储 XML 文件信息类
2. `XmlDeclaration` 用于添加版本信息
3. `XmlNode` 代表每一个节点对象

- XML 文件存储流程示例
为什么将 XML 文件保存在 `persistentDataPath` 中？
因为只有该文件在打包后找得到路径，并且还可读可写，详见 [[学习笔记/编程/Unity/Unity四部曲/基础/知识点笔记#特殊文件夹|特殊文件夹]]
```C#
// 1.创建文本对象
XmlDocument xml = new XmlDocument();

// 2.添加固定版本信息
// 相当于创建 <?xml version="1.0" encoding="UTF-8"?> 这句内容
// 参数三不用管 填空字符串即可
XmlDeclaration xmlDec = xml.CreateXmlDeclaration("1.0", "UTF-8", "");
xml.AppendChild(xmlDec);

// 3.编写文件的 XML 格式节点信息
// 添加根节点
XmlElement root = xml.CreateElement("Root");
xml.AppendChild(root);
// 添加子节点(对元素节点)
XmlElement name = xml.CreateElement("name");
// 设置子节点的内容
name.InnerText = "曹哲";
root.AppendChild(name);
// 添加复杂的子节点
XmlElement itemList = xml.CreateElement("itemList");
for (int i = 1; i <= 3; i++)
{
    XmlElement childNode = xml.CreateElement("Item");
    // 添加子节点(对属性)
    childNode.SetAttribute("id", i.ToString());
    childNode.SetAttribute("num", (i * 10).ToString());
    itemList.AppendChild(childNode);
}
root.AppendChild(itemList);

// 4.调用 API 将文件保存到合适路径
string path = Application.persistentDataPath + "/PlayerInfo2.xml";
xml.Save(path);
```

- XML 文件的修改
```C#
// 先通过路径判断 XML 文件是否存在
if( File.Exists(path) )
{
	// 再加载要修改的 XML 文件
	XmlDocument newXml = new XmlDocument();

	// 想要移除节点内容，需要获取父和子节点对象，通过父节点删除子节点
	XmlNode root = newXml.SelectSingleNode("Root");
	XmlNode node = newXml.SelectSingleNode("Root/atk");
	// 移除子节点
	root.RemoveChild(node);
	// 添加子节点
	XmlElement speed = newXml.CreateElement("moveSpeed");
	speed.InnerText = "20";
	root2.AppendChild(speed);
	
	// 修改完毕后要进行保存
	// 相当于把原来的文件覆盖掉了
	newXml.Save(path);
}
```

## XML 序列化和反序列化

- 实现序列化和反序列化的关键
1. `XmlSerializer` 用于序列化对象为 XML 的类（翻译机器）
2. `StreamWriter` 用于存储文件的类（翻译后的文件去路）
3. `using` 用于方便流对象释放和销毁

- 注意
这种方式不支持字典的序列化和反序列化，只能对公开的成员进行序列化和反序列化
但可以通过 `IXmlSerializable` 这个接口自定义一个类的序列化和反序列化规则

### 序列化
把对象转化为可传输的字节序列过程

- 示例
```C#
// 测试类对象
TestClass test = new TestClass();

// 将对象序列化为 XML 文件后的存放路径
string path = Application.persistentDataPath + "/Test.xml";

// using 括号包裹的声明对象会在大括号语句块结束后 自动释放掉
// 一般都是配合内存占用比较大或者有读写操作时进行使用
using (StreamWriter stream = new StreamWriter(path))
{
	// 将对象 TestClass 序列化为 XML 文件
    XmlSerializer s = new XmlSerializer(typeof(TestClass));
    // 通过序列化对象将类对象进行翻译 将其翻译成 XML 文件 写入到对应的文件中
    // 参数一：文件流对象，翻译后的 XML 文件存入文件流对象的路径内
	// 参数二：想要被翻译的对象
	// 注意：翻译机器的类型 一定要和被翻译的对象是一致的 不然会报错
	// 序列化时 如果对象中的引用成员为空 则 XML 文件里面是看不到该字段
    s.Serialize(stream, test);
}
```

- 小技巧
通过特性，来设置序列化后 XML 文件中的节点名字或者设置为属性
```C#
class TestClass
{
	// 加上该特性的成员被序列化后将以属性存在而不是元素
	// 括号内可填写字符串，表示该节点的名字
	[XmlAttribute("Test")]
	public int num;
	
	// 加上该特性的成员被序列化后将以元素存在而不是属性
	// 括号内可填写字符串，表示该节点的名字
	[XmlElement("Test")]
	public string name;
	
	// 该特性设置容器元素的名称
	[XmlArray("IntList")]
	// 该特性设置容器内成员的元素的名称
	[XmlArrayItem("Int32")]
	public List<int> listInt;
}
```

### 反序列化
把字节序列还原为对象的过程

- 示例
```C#
// 测试类对象
TestClass test = new TestClass();

// 获取 XML 文件的路径
string path = Application.persistentDataPath + "/Test.xml";

// 判断是否存在该文件
if( File.Exists(path) )
{
    using (StreamReader stream = new StreamReader(path))
    {
        XmlSerializer s = new XmlSerializer(typeof(Test));
        // 从 stream 文件流中读取 XML 文件反序列翻译出来转成 Test
        Lesson1Test lt = s.Deserialize(stream) as Test;
    }
}
```

### 自定义类的序列化和反序列化规则

要进行序列化和反序列化的类通过继承 `IXmlSerializable` 接口
将得到如下方法：
```C#
public class Test : IXmlSerializable  
{
	public int test1;
	public string test2;
	
    public XmlSchema GetSchema()
    {
	    // 这里固定返回空就行
	    return null;
    }
    
    public void ReadXml(XmlReader reader)
    {
	    // 这里写反序列化的自定义规则
    }
	
    public void WriteXml(XmlWriter writer)
    {
		// 这里写序列化的自定义规则
	}
}
```

- 自定义序列化规则的方法
```C#
// 这里写反序列化的自定义规则
public void ReadXml(XmlReader reader)
{
	// 1.读属性
	this.test1 = int.Parse(reader["test1"]);
	this.test2 = reader["test2"];
	
	// 2.读节点
	// 方式一
	reader.Read(); // 这时是读到的test1节点
	reader.Read(); // 这时是读到的test1节点包裹的内容
	this.test1 = int.Parse(reader.Value); // 得到当前内容的值
	reader.Read(); // 这时读到的是尾部包裹节点
	reader.Read(); // 这时是读到的test2节点
	reader.Read(); // 这时是读到的test2节点包裹的内容
	this.test2 = reader.Value;
	// 方式二
	while(reader.Read())
	{
	    if( reader.NodeType == XmlNodeType.Element )
	    {
		    switch (reader.Name)
	        {
		        case "test1":
	                reader.Read();
	                this.test1 = int.Parse(reader.Value);
	                break;
	            case "test2":
	                reader.Read();
	                this.test2 = reader.Value;
	                break;
	        }
	    }
	}
	
	// 3.读包裹元素节点(子节点)
	XmlSerializer s = new XmlSerializer(typeof(int));
	XmlSerializer s2 = new XmlSerializer(typeof(string));
	// 跳过根节点
	reader.Read();
	reader.ReadStartElement("test1");
	test1 = (int)s.Deserialize(reader);
	reader.ReadEndElement();
	
	reader.ReadStartElement("test2");
	test2 = s2.Deserialize(reader).ToString();
	reader.ReadEndElement();
}

// 这里写序列化的自定义规则
public void WriteXml(XmlWriter writer)
{
	// 1.写属性
	writer.WriteAttributeString("test1", this.test1.ToString());
	writer.WriteAttributeString("test2", this.test2);
	
	// 2.写节点
	writer.WriteElementString("test1", this.test1.ToString());
	writer.WriteElementString("test2", this.test2);
	
	// 3.写包裹节点(子节点)
	XmlSerializer s = new XmlSerializer(typeof(int));
	writer.WriteStartElement("test1");
	s.Serialize(writer, test1);
	writer.WriteEndElement();
	
	XmlSerializer s2 = new XmlSerializer(typeof(string));
	writer.WriteStartElement("test2");
	s2.Serialize(writer, test2);
	writer.WriteEndElement();
}
```

- 自定义字典 XML 序列化和反序列规则
```C#
public class SerizlizerDictionary<TKey, TValue> : Dictionary<TKey, TValue>, IXmlSerializable
{
    public XmlSchema GetSchema()
    {
	    return null;
    }
    // 这里写反序列化的自定义规则
    public void ReadXml(XmlReader reader)
    {
	    XmlSerializer keySer = new XmlSerializer(typeof(TKey));
        XmlSerializer valueSer = new XmlSerializer(typeof(TValue));
		
        // 要跳过根节点
        reader.Read();
        while (reader.NodeType != XmlNodeType.EndElement)
        {
	        // 反序列化键
            TKey key = (TKey)keySer.Deserialize(reader);
            // 反序列化值
            TValue value = (TValue)valueSer.Deserialize(reader);
            // 存储到字典中
            this.Add(key, value);
        }
    }
    // 这里写序列化的自定义规则
    public void WriteXml(XmlWriter writer)
    {
	    XmlSerializer keySer = new XmlSerializer(typeof(TKey));
        XmlSerializer valueSer = new XmlSerializer(typeof(TValue));
		
        foreach (KeyValuePair<TKey, TValue> kv in this)
        {
	        // 键值对的序列化
            keySer.Serialize(writer, kv.Key);
            valueSer.Serialize(writer, kv.Value);
        }
    }
}
```
