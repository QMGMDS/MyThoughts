# 3D数学

## 数学计算公共类 Mathf
- 概述
`Math` 是C#中封装好的用于数学计算的工具类 -> 位于 `System` 命名空间中
`Mathf` 是Unity中封装好的用于数学计算的工具结构体 -> 位于 `UnityEngine` 命名空间中
二者都是为了方便数学相关计算的
`Mathf` 是 Unity 专门封装的，不仅包含 `Math` 中的方法，还多了一些适用于游戏开发的方法
因此使用 `Mathf` 中的方法用于数学计算即可

- 计算一次的常用方法
1. π
```C#

```
2. 取绝对值

```

```
6. 获取最大值

- 计算多次的常用方法
`Lerp` 插值运算
函数公式：`result = Mathf.Lerp(start, end, t)`
其中 t 为插值系数，取值范围为 0~1，等价于下面的运算公式：
运算公式：`result = start + (end - start)*t`

1. 插值运算用法一
```C#
private float start = 0f; // 变

void Update()
{
	// 每帧改变start的值 -> 趋近速度先快后慢，位置无限接近，但是永远不会得到end位置
	start = Mathf.Lerp(start, 10, Time.deltaTime);
}
```

2. 插值运算用法二
```C#
private float start = 0f; // 不变
private float result = 0f;
private float time = 0f;

void Update()
{
	// 每帧改变t的值 -> 趋近速度匀速，位置每帧接近，当t>=1时，得到结果
	time += Time.deltaTime;
	result = Mathf.Lerp(start, 10, time);
}
```

- 向量点乘判断对象相对自身方位（前方和后方）
```C#
// 根据点乘结果判断目标物体在自身方位的前方还是后方
// 两个向量是否是单位向量对方位判断结果无影响
// this.transform.forward 即 自身方位的正前方
// target.position 即 目标物体的坐标
float dotResult = Vector3.Dot(this.transform.forward, target.position - this.transform.position);

if( dotResult >= 0 )
{  
    print("它在我前方");  
}  
else  
{  
    print("它在我后方");  
}
```
示意图：
左边物体 A 即自身，右边物体 B 为目标
$$
\vec{a} \cdot \vec{b} = |\vec{a}| \cdot |\vec{b}| \cdot \cos \theta
$$
其中 $|\vec{b}| \cdot \cos \theta$ 就是 b 向量在 a 向量方向上的投影向量的模长
点积的结果为 >0 则说明 B 在 A 的前方
点积的结果为 =0 则说明 B 在 A 正左侧或正右侧
点积的结果为 <0 则说明 B 在 A 的后方
就拿两向量点乘的结果本身没有几何意义，但通过结果正负判断有意义

- 向量点乘推导公式算出对象相对自身夹角（扇形）
```C#
// 用单位向量算出点乘结果 -> 夹角余弦值
float dotResult = Vector3.Dot(this.transform.forward, (target.position - this.transform.position).normalized);
// 用反三角函数得出弧度 随后转角度
print(Mathf.Acos(dotResult) * Mathf.Rad2Deg); // 夹角角度

// Vector3 中也提供了直接得到两个向量之间夹角的方法
// 传入要算夹角的两个向量，是否是单位向量对结果无影响
print(Vector3.Angle(this.transform.forward, target.position - this.transform.position)); // 夹角角度
```
$$
\cos \theta = \frac{\vec{a} \cdot \vec{b}}{|\vec{a}| \cdot |\vec{b}|}
$$
也可以看作两个方向的单位向量的点乘
夹角余弦值 -> 反余弦函数获得弧度 -> 弧度转角度 -> 两物体之间的夹角 (0~180°)
0~90° 代表 B 物体在 A 的前方
90~180° 代表 B 物体在 A 的后方 

- 向量叉乘判断对象相对自身方位（左侧和右侧）
```C#
// OA叉乘OB
// O为坐标原点
Vector3 C = Vector3.Cross(A.position, B.position);
if( C.y > 0)
{
    print("B在A的右侧");
}
else  
{
    print("B在A的左侧");
}
```
两向量叉乘的结果是得到一个垂直于两向量组成平面的新向量
对于 $\vec{a} \times \vec{b}$ ：
新向量的 y 值 > 0 -> 物体B在A的右侧
新向量的 y 值 < 0 -> 物体B在A的左侧
新向量的 y 值 = 0 -> 物体B在A的正前方或正后方
注意：叉乘不具有交换律
$$
\begin{align}
\vec{a} \times \vec{b} = \vec{C} \\
\vec{b} \times \vec{a} = \vec{D} \\
\vec{C} = -\vec{D}
\end{align}
$$

- Vector3 的线性插值
```C#
public Transform A;
public Transform target;

private float time;
private Vector3 startPos;
private Vector3 nowTarget;

void Start()
{
	startPos = this.transform.position;
}

void Update()
{
	// 公式
    // result = start + (end - start) * t
    // Vector3.Lerp(start, end, t);
    
    // 先快后慢：每帧改变start位置 位置无限接近 但不会得到end位置  
    A.position = Vector3.Lerp(A.position, target.position, Time.deltaTime);
    
    // 匀速：每帧改变时间 当t>=1时得到结果
    if(nowTarget != target.position)
    {
	    nowTarget = target.position;
        time = 0;
        startPos = A.position;
    }
    time += Time.deltaTime;
    B.position = Vector3.Lerp(startPos, nowTarget, time);
}
```

## Quaternion 四元数
像欧拉角一样，四元数也是表示三维空间中物体旋转信息的一种方式

- 为什么使用四元数？
原因是使用欧拉角表示物体的旋转存在一些缺点：
1. 同一旋转结果的表示不唯一，即不同旋转路径能达到一样的最终效果
2. 万向节死锁
万向节死锁即：当某个特定轴达到某个特殊值时，绕一个轴旋转可能会覆盖住另一个轴的旋转从而失去一维自由度，比如 Unity 中 x 轴达到 90° 时会产生万向节死锁现象
而四元数旋转不存在万向节死锁问题
因此在计算机中往往使用四元数来表示三维空间中的旋转信息

- 概述
四元数的本质是描述三维空间中物体旋转信息的一种方法
通过 Unity 封装好的 `Quaternion` 结构体，可以解决欧拉角表示物体旋转信息时产生的问题

对于给定旋转，假设为绕着 $n$ 轴，旋转 $\beta$ 度，$n$ 轴向量表示为 $(x,y,z)$ 那么可以构成四元数为：
$$Q = (cos(\frac{\beta}{2}),  sin(\frac{\beta}{2})x, sin(\frac{\beta}{2})y, sin(\frac{\beta}{2})z)$$
