# 最开始的配置（项目升级URP）

1. 创建项目
	新建一个 `3D (Built-In Render Pipeline) Core` 为模板的项目
	选用内置渲染管线是为了学习如何自行升级成 URP

2. 项目升级到 URP
	在 `Window/Package Manager` 中 `Unity Registry` 中搜索 `Universal RP` 并安装
	随后在 Project 窗口中创建一个新的 `UniversalRenderPipelineAsset`，并打开 `Edit/Project Settings`，里面 `Graphics` 配置 `Scriptable Render Pipeline Settings`，`Quality` 配置 `Rendering`，两者都选中刚刚创建好的 `PipelineAsset`
	这里创建出来的 `UniversalRenderPipelineAsset` 和自动创建出来的 `UniversalRenderPipelineAsset_Renderer` 建议放在同一的文件夹 `Assets/Pipeline Settings` 中统一管理 
	通过修改 `UniversalRenderPipelineAsset` 在 `Inspector` 中的参数可以修改一些渲染设置 [[#管理 UniversalRenderPipelineAsset]]

3. 素材适配当前 URP 通用渲染管线
	导入素材后（注意：导入的素材要支持 URP）
	此时发现将素材拖入场景中显示有问题
	这是因为素材未进行当前渲染管线的适配
	选择 `Edit/Render Pipeline/Upgrade Projct Materials to UniversalRP Materials` 将整个项目的 Materials 都升级到 URP
	此时素材正常显示

4. 场景光照渲染设置
	选择 `Window/Rendering/Lighting` 后出现窗口
	Lighting 窗口中 Scene 里面选择 `New Lighting Settings` 创建一个灯光设置的配置文件
	将文件同一放在 `Pipeline Settings` 文件夹中
	像这样设置：
	![[Pasted image 20260305205841.png]]
	完成后点击 `Generate Lighting` 即可
	更多参数详情：[[#Lighting 窗口]]

## Shadows

![[Pasted image 20260305204425.png]]

- Max Distance
	修改该项数值控制场景中物体渲染阴影的范围
	比如：设为50 则在50米内才去渲染阴影 超出范围则物体不显示阴影

- Cascade Count
	控制阴影显示的虚实级别
	比如：该值改为2时 距离物体 `Split 1(又一个参数)` 米时显示清楚的阴影
	`Split 1 ~ Max Distance` 两个参数的范围内显示模糊的阴影
	Cascade Count的值设置为3等以此类推

- Soft Shadows
	影响阴影显示效果 开启阴影就会变得虚虚的感觉

## PolyBrush

方便在 3D 项目中绘制场景的插件

- 配置：`Panckage Manager` 中安装 `Unity Registry` 中的 `PolyBrush`

- 在 Unity 编辑器工具栏中 `Tool/Polybrush/Polybrush Window` 下打开 Polybrush 窗口
	![[Pasted image 20260309193659.png]]
	其中：
	- `Outer Radius`：作用范围的外圆半径
	- `Inner Radius`：作用范围的内圆半径
	- `Strength`：作用强度
	- `Brush Mirroring`：以目标物体中心点为对称中心 沿 xyz 轴方向进行镜像刷取
	- `World/Camera`：不懂，尽量选择 World
	
	选择这些工具后点击对应的目标物体 物体上会被分割成许多多边形
	这些多边形的拼接在一起 而多边形的顶点才是能去绘制的目标
	比如：增高地面 只能对顶点所在位置为中心以给定半径的范围进行内外两层不同强度起伏进行绘制
	
	
	1. 调整场景中物体的高度起伏（例如地面）
		- `Direction`：枚举变量 控制目标物体的拉伸方向
		默认点击拖拽向正方向拉伸物体
		按住Ctrl点击拖拽向负方向拉伸物体
	
	2. 柔化
		将一些起伏较大的地方进行柔和处理
	
	3. 对场景中物体刷上不同颜色
		注意：使用该工具需要让目标物体的材质是 Polybrush 材质否则刷取时无法正常显示出来
		如何正确改变物体的材质？
		![[Pasted image 20260309201710.png]]
		在安装完 Polybrush 时给出了三个 Sample
		如果你的项目升级成 URP 则导入第二个
		没有升级则导入第一个
		![[Pasted image 20260309201914.png]]
		此时文件夹中会出现两个提供的材质 如图创建 Color 的 Material 材质 生成材质文件 建议将所有的材质文件统一放到文件夹中管理
		![[Pasted image 20260309202101.png]]
		随后将该材质应用在目标物体上（直接拖拽到物体身上）
		提示：在物体的 Inspector 窗口中有 Material 材质的组件 修改材质就是修改改组件的内容
		- `Brush/Fill/Flood`：表示笔刷 多边形范围 全部 三个刷取范围
		- `Color Mask`：要刷取的颜色
	
	4. 将预制体批量刷取到场景中
		将要刷取的预制体拖到 `Current Palette` 调色盘中
		随后在调色盘中选中要刷取的预制体(打钩后会自动添加到 `Brush Loadout`中) 调整被选中预制体的滑动条改变其刷取时出现概率
		按住Ctrl键再进行点击将会删除目标预制体(前提这个预制体是被刷上去的)
		
		- `Hit Surface is Parent`：是否将刷取的预制体作为选中目标物体子级

可以发现
在物体上进行刷取的精度是根据物体上多边形的顶点数目决定的
如果想扩大物体（比如说游戏的整个地面）
通过修改物体的缩放 `Transform/Scale` 虽然可以让物体变大 但是物体多边形顶点的数目不变 即通过修改缩放来扩大物体的方式会损失刷取的精度

# 地图烘焙

通过该工具 可以设置人物移动区域
相当于给人物和场景分别设置碰撞体
而场景分为可行走碰撞体和不可行走碰撞体
预先设置好场景上所有物体是否可行走的参数后
在 Navigation 窗口 Bake 中进行烘焙即可将可行走区域（淡蓝色区域）显示出来 没有淡蓝色的地方就是不可行走的区域

- 在 Unity 编辑器工具栏中 `Windows/AI/Navigation` 下打开 Navigation 窗口
	- `Agent`
	- `Areas`
	- `Bake`
		执行修改 对所有 `Navigation Static` 物体进行烘焙 `Max Slope` 代表该物体可移动的最大坡度
		- `Agent Radius`：物体圆柱体的半径
		- `Agent Height`：物体圆柱体的高度
		这两个参数构成的圆柱体将成为 Navigation 判断玩家大小 生成适合玩家的场景移动范围 最好参数设置成和玩家圆柱体相同
		
	- `Object`
		选中物体后 其中 `Navigation Area` 代表该物体是否可以被人物移动（想要修改此项需要先将目标物体设置为 `Navigation Static`）

- 设置可行走区域
	将地面设置为 `Navigation Static` 后在 `Object` 中设置为 `Walkable` 随后 Scene 场景中显示淡蓝色的区域即人物可行走的区域

- 设置不可行走区域
	- 方式一：
		面板上设置(被设置的物体在游戏运行时静止)
		选中不可行走的物体 将其设置为 `Navigation Static`  随后在 `Object` 中设置为 `Not Walkable` 烘焙后该物体周围区域将不显示淡蓝色
		注意：该方式将时物体变成静态 如果想要一个可移动的平台但是人物由与之碰撞不会穿模则需要选择方式二
	- 方式二：
		添加组件(被设置的物体在游戏运行时可移动)
		为目标物体添加 `Nav Mesh Obstacle` 组件
		- `Carve`：勾选该选项将在打开 Navigation 后实时显示该物体的障碍范围

- 设置目标人物的 Navigation 
	在人物物体身上添加 `Nav Mesh Agent` 组件 配合场景物体的Navigation 实现人物与场景移动的交互 相当于两个碰撞体
	- `Radius`：人物圆柱体的半径
	- `Height`：人物圆柱体的高度
	- 更多参数说明：[[#^066ba8]]

## 摄像机跟随人物移动

通过 Cinemachine 中提供的方法实现摄像机跟随人物移动

- 配置：
	`Panckage Manager` 中安装 `Unity Registry` 中的 `Cinemachine`
	在 Unity 编辑器工具栏中 `Cinemachine/Create Virtual Camera` 创建一个虚拟摄像机 随后场景中多了个 CM vcam1 作为 Main Camera 的大脑(  

- 初始设置虚拟相机
	`Body` 改为 `Framing Transposer`
	`Anim` 改为 `Do nothing`
	![[Pasted image 20260310213000.png]]
	认识一些参数：
	`Follow`：摄像机要跟随的物体
	`Dead Zone Width`：缓冲宽度
	`Dead Zone Height`：缓冲高度

- 优化 点击位置
	我们注意到 当点击场景中的物体(比如树木)时 由于鼠标的射线被物体遮挡 该物体又不是我们设置的 Ground 的 Tag 所以人物在建筑物体背后则无法移动
	以下是解决方法
	将所有场景中装饰物体的 Layer 改为 Ignore Raycast
	这样人物在物体背后也能移动了

则会执行该方法获得一个基于最开始敌人放置位置为中心 在巡逻范围内的随机点 `watPoint`
随后赋值给 `agent.destination` 实现敌人移动
当敌人当前位置与巡逻点位置距离小于 `agent.stoppingDistance` 时再去随机一个点 循环往复

需要注意的时 如果随机到一个阻挡物的点 即在 Navigation 中是 Not Walkable 的点
敌人则会由于无法移动到该位置而卡住
解决方法是 在生成随机点后判断该点是否可行走
通过 Navigation 提供的方法搜索随机点区域范围内可行走区域的点并返回
如果没有则返回敌人当前坐标
在 `SwitchStates()` 中会判为走到巡逻位置
于是重新生成一个随机巡逻点
