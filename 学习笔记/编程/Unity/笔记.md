- 综合模式下：

# 预制体

- 预制体是什么？
	预制体指预先制作的好的物体在 Project 窗口中的显示
	预制体不仅会保留物体父子层级的关系 还会保存所有的脚本信息
	将创建的物体保存为预制体 能方便快速的实现复用而不需要重复操作

- 预制体的创建
	将想要做成的预制体的物体直接拖入 Project 窗口中即可
	![[Pasted image 20260307142809.png]]

- 预制体的修改
	通过双击 Project 窗口中的目标预制体进入预制体编辑界面
	或者点击场景中你想要修改的预制体右边 “>” 箭头进入预制体编辑界面
	在预制体编辑界面对预制体做出的修改都会实时影响到场景中已有的与之相关联的预制体

- 若要临时修改预制体
	如果我已经对场景中预制体做出修改
	现在要将它关联的预制体更新场景当前做出的更改
	但又不想进入预制体编辑模式更新
	- 如图：
		选中场景中的预制体后
		Overrides 中选择 Apply All
		表示更新已保存的预制体信息为当前场景中预制体对象当前的修改
		而 Revert All 表示重置当前选中的预制体关联对象为已保存的预制体
		![[Pasted image 20260307143443.png]]
	- 强制覆盖
		直接将场景中预制体拖入对应 Project 窗口中保存的预制体位置进行覆盖

- 预制体的删除
	直接删除 Project 窗口中的预制体对象即可
	但场景中关联该预制体的对象会报出红色显示
	像这样：![[Pasted image 20260307144425.png]]
	这表示该对象先前关联过预制体 但被删除了
	虽然不影响使用但是看着不舒服
	可以右键该对象：![[Pasted image 20260307144637.png]]
	选择 Unpack Prefab 后即可取消该对象的预制体关联
	这样它就变成了普通的对象

# Unity 脚本基础
## 脚本基本规则

- 脚本的注意事项：
	1. Unity 工程创建的所有脚本统一放在 `Assets/Scripts` 文件夹中管理
	2. 创建脚本中的类名和文件名必须一致 不然不能挂载（因为反射机制创建对象 会通过文件名去找 Type)
	3. 不要使用中文名命名脚本
	4. 创建的脚本会默认继承 MonoBehavior

- MonoBehavior 基类：
	继承 Mono 的脚本就是为了挂在物体身上而存在的
	1. 继承了 MonoBehaviour 的脚本才能够被挂载在 GameObject 上
	2. 继承了 MonoBehavior 的脚本不能 new 只能挂
	3. 继承了 MonnBehavior 的脚本不要去写构造函数 因为不会去 new 它 写构造函数没有任何意义
	4. 继承了 MonoBehavior 的脚本可以在一个对象上挂多个（如果没有加 DisallowMultipleComponent 特性 加了该特性的脚本则无法在同一个物体上挂载多个该脚本）
	5. 继承 MonoBehavior 的类也可以再次被继承 遵循面向对象继承多态的规则

- 不继承 Mono 的类
	1. 不能挂载在 GameObject 上
	2. 允许 new 一个对象出来 new 出来才能使用
	3. 一般是单例模式的类(用于管理模块)或者数据结构类(用于存储数据)
	4. 不要使用生命周期函数 也没有意义

- 脚本的执行顺序
	在 `Project Settings/Script Execution Order` 中显示的就是脚本执行顺序
	可以自行添加要执行的脚本 数字越大越后执行
	但是挂载在多个对象的同一个脚本的执行顺序无法控制
	![[Pasted image 20260307204524.png]]

- 默认脚本内容
	在安装的 Unity 对应版本引擎的 `Editor\Data\Resources\ScriptTemplates` 文件目录中
	每次新创建的脚本初始内容文件就是这个
	通过修改该文件可以改变初始新脚本的内容
	![[Pasted image 20260307204942.png]]

## 生命周期函数

所有继承 MonoBehavior 的脚本最终都会挂载到 Gameobject 游戏对象上
生命周期函数就是该脚本对象依附的 Gameobject 对象从出生到消亡整个生命周期中
会通过反射自动调用的一些特殊函数
Unity 帮助我们记录了一个 Gameobject 对象依附了哪些脚本
通过反射去执行一些固定名字的函数

生命周期函数的访问修饰符一般为 `private` 或 `protected`
因为不需要外部去调用自己的生命周期函数 是 Unity 自己调用的

如果你不需要再生命周期函数中写东西
那你就不要去写生命周期函数
这会导致一部分性能开销

# 必不可少的 Camera

## 认识 Camera 组件可编辑参数

*加粗的为重要内容*

- **Clear Flags**
	- `Skybox` 天空盒子 搭配 `Background` 改变盒子颜色
	- `Solid Color` 背景纯颜色填充 搭配 `Background` 改变填充颜色
	- `Depth only` 只画该层 背景透明 搭配 `Depth` 来实现两个摄像机渲染内容叠加在一起的效果
	- `Don't Clear` 不移除 覆盖渲染 (用的很少 不需要了解)

- **Culling Mask**
	选择性渲染部分层级 可以指定摄像机只渲染对应层级的对象
	对应 Layer 中的所有层级

- **Projection**
	- `Perspective` 透视模式 人眼看东西的模式
		- `FOV Axis` 一般默认 Vertical
		- `Field of View` 视口大小 类似 mc 的超广角
		- `Physical Camera` 模拟现实世界物理摄像机 勾选后会出现物理摄像机参数 焦距 传感器尺寸 透镜位移等
	
	- `Orthographic` 正交模式 一般2D游戏使用的模式
		- `Size` 摄制范围

- **Clipping Planes**
	裁剪平面距离
	- `Near` 摄像机最近能看到的距离 数值越大越近的物体越看不到
	- `Far` 摄像机最远能看到的距离 数值越小越远的物体越看不到

- Viewport Rect
	改变视口范围 屏幕将显示该摄像机的视图位置
	主要用于双摄像机游戏 像是游戏双人模式一人占用一半屏幕
	0~1相当于宽高百分比

- **Depth**
	渲染顺序上的深度
	值越小表明该摄像机拍摄的内容越先被渲染(相当于第一张照片)
	而值越大的摄像机由于后面渲染出来东西所以会将先前的东西覆盖(相当于第二张照片盖住了第一张照片)
	
	搭配 Clear Flags 的 `Don't Clear` 模式使用：
	比如将第二摄像机模式设置为 `Don't Clear` 则该摄像机只会渲染要渲染的物体 而不会渲染背景（将背景透明）
	由于第二摄像机的 Death 更大 所以它覆盖了第一摄像机的内容 但是又由于背景透明 所以表现效果是两个摄像机渲染的内容被叠加了

- Redering Path
	渲染路径
	默认不去修改 使用 `Use Graphics Settings` 就行

- **Target Texture**
	渲染纹理
	Project 窗口中右键可以创建一个 Render Texture 
	将该文件拖拽赋值进来
	则摄像机渲染的内容全部显示在该文件中
	相当于将摄像机的画面渲染到一张图上（主要用于制作小地图）

- **Occlusion Culling**
	是否启用剔除遮挡
	开启后将不会渲染场景中被其他物体遮挡住的物体 节约性能开销
	一般默认勾选

- Allow HDR
	是否允许高动态范围渲染

- Allow MSAA
	是否允许抗锯齿

- Allow Dynamic Resolution
	是否允许动态分辨率呈现

- Target Display
	该摄像机用于哪个显示器
	主要用于开发有多个屏幕的平台游戏

# 核心系统

## 光源系统基础

### 认识 Light 组件可编辑参数

*加粗的为重要内容*

- **Type**
	光源类型
	- `Spot` 聚光灯
		`Range` 发光范围距离
		`Spot Angle` 光锥角度
	- `Directional` 方向光(环境光)
	- `Point` 点光源
	- `Area` 面光源(仅在烘焙后有效果)

- **Color**
	光的颜色

- **Mode**
	光源模式
	`Realtime` 实时光源 每帧实时计算 效果好 性能消耗大
	`Baked` 烘焙光源 事先计算好 无法动态变化
	`Mixed` 混合光源 预先计算+实时运算

- **Intensity**
	光源亮度

- Indirect Multiplier
	改变间接光的强度
	低于1 每次反弹会使光更暗
	大于1 每次反弹会使光更亮

- **Shadow Type**
	对当前光照射到的物体的阴影处理
	`No Shadows` 关闭阴影
	`Hard shadows` 生硬阴影
	`Soft shadows` 柔和阴影

- RealtimeShadows
	- `Strength` 阴影暗度0~1之间，越大越黑
	- `Resolution` 阴影贴图渲染分辨率，越高越逼真，消耗越高
	- `Bias` 阴影推离光源的距离
	- `Normal Bias` 阴影投射面沿法线收缩距离
	- `Near Panel` 渲染阴影的近裁剪面

- **Cookie**
	投影遮罩
	放入遮罩的图片资源(资源后缀为.tif) 遮罩图片由黑白两色组成
	相当于给光线上了一层过滤片
	透过镜片的光线显示在场景中
	图片资源黑色的地方表示光被遮挡
	白色的地方表示光没有被遮挡

- Cookie Size
	只有方向光才有该参数
	投影遮罩的大小

- **Draw Hab**
	光源的特殊效果 球形光环开关 开了相当于光晕效果

- **Flare**
	光源的特殊效果 耀斑
	将 .flare 资源拖入
	如果要在 Game 场景中显示耀斑效果
	需要为 Main Camera 摄像机添加脚本 Flare Layer

- Render Mode
	渲染优先级
	`Auto` 运行时确定
	`Important` 以像素质量为单位进行渲染 效果逼真 消耗大
	`Not Important` 以快速模式进行渲染

- **Culling Mask**
	剔除遮罩层
	决定哪些层的对象收到该光源影响

### 光面板相关

- 光面板打开
	Unity 编辑器工具栏 `Window/Rendering/Lighting Settings` 打开光设置面板

Scene 中：

- Environment 环境相关设置
	- `Skybox Material` 天空盒材质 修改改变天空盒
	- `Sun Source` 太阳来源 不设置会默认使用场景中最亮的方向光代表太阳
	- `Environment Lighting` 环境光设置
		- `Source` 环境光光源颜色
		- `Intensity Multiplier` 环境光亮度
		- `Ambient Mode` 全局光照模式 只有启用了实时全局和全局烘焙时才有用

- Other Settings 其它设置
	- `Fog` 雾开关
		- `Color` 雾颜色
		- `Mode` 雾计算模式
			- `Linear` 随距离线性增加
				`Start` 离摄像机多远开始有雾
				`End` 离摄像机多远完全遮挡
			- `Exponential` 随距离指数增加
				`Density` 强度
			- `Exponential Qquare` 随距离比指数更快的增加
				`Density` 强度
	- `Halo Texture` 光源周围挥着光环的纹理
	- `Halo Strength` 光环可见性
	- `Flare Fade Speed` 耀斑淡出时间 最初出现之后淡出的时间
	- `Flare Strength` 耀斑可见性
	- `Spot Cookie` 聚光灯剪影纹理
