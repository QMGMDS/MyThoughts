
> [!summary] 总结
C#中所有参数默认都按值传递，但对于不同类型的含义不同
>
**值类型参数**
>- 传递的是变量的**副本**。    
>- 方法内修改参数不会影响原始变量。  
 >
**引用类型参数**
>- 传递的是**引用的副本**（即内存地址的副本）。
>- 方法内可以修改**对象的内容**，因为副本引用指向同一个对象。      
>- 但不能让调用者的变量指向新对象（因为传的是引用的副本，不是引用本身）。
>
>想要改变这种行为，必须显式使用 `ref`、`out` 或 `in` 关键字。这是C#精心设计的安全特性，防止意外修改数据。

# 按值类型传递

**传递的是变量的副本**

示例#1：
``` C#
void ModifyValue(int x)
{
    x = 100;  // 只修改副本，不影响原始值
}

int num = 10;
ModifyValue(num);
Console.WriteLine(num);  // 输出：10
```

示例#2：
``` C#
class Person
{
    public string Name;
}

void ModifyObject(Person p)
{
    p.Name = "Changed";  // 修改的是原始对象
}

var person = new Person { Name = "Original" };
ModifyObject(person);
Console.WriteLine(person.Name);  // 输出："Changed"
```

注意：由于[参数是引用类型](参数类型.md)的特点，可以通过副本修改对象内容，但不能通过副本改变调用者的引用


# 按引用类型传递

**传递的是对象的引用（内存地址）**


示例#1：
``` C#
class Person
{
    public string Name;
}

void ModifyObjectRef(ref Person p)
{
    p = new Person { Name = "New" };  // 这会改变原始 person 引用
}

var person = new Person { Name = "Original" };
ModifyObjectRef(ref person);
Console.WriteLine(person.Name);  // 输出 "New"
```
