## 头文件的引用说明

```c++
// 用 <> 包裹起来的是引用系统提供的写好的代码，编译器会在专门的系统路径中去查找
#include <iostream>
// 用 "" 包裹起来的是引用我们自己写到的代码，编译器会在当前文件所在目录中查找
// 找不到才会去系统路径重查找
#include "myscripts"
```

## 折叠代码

```c++
#pragma region 折叠内容描述
// 这里写被折叠起来的代码和注释
// 折叠代码只会在编辑时有用，发布代码或执行代码时会被自动删除
#pragma endregion
```

# 变量

14 种基本类型：

- 特殊类型
	- `bool` 布尔型，取值为 `true` 或 `false` 
	- `char` 单个字符，C++中能被当做整型对待，字符对应的数值就是它的 ASCII 码值
	- `string` 多个字符，没有存储上限
- 整数类型
	- 有符号整型
		- `short` 有符号短整型，存储范围：-32768 ~ 32767
		- `int` 有符号整形，存储范围：-21亿多 ~ 21亿多
		- `long` 有符号长整型，存储范围：-21亿多 ~ 21亿多 或 -9223亿亿多 ~ 9223亿亿多
		- `long long` 有符号长长整型，存储范围：-9223亿亿多 ~ 9223亿亿多
	- 无符号整型
		- `unsigned short` 无符号短整型，存储范围：0 ~ 65535
		- `unsigned int` 无符号整形，存储范围：0 ~ 42亿多
		- `unsigned long` 无符号长整型，存储范围：0 ~ 42亿多 或 0 ~ 18446亿亿多
		- `unsigned long long` 无符号长长整型，存储范围：0 ~ 18446亿亿多
- 浮点类型
	- `float` 单精度浮点数，存储范围：-3.4* 10^-38 ~ 3.4* 10^38 ，存储 6/7 位有效数字
	- `double` 双精度浮点数，存储范围：-1.7* 10^-308 ~ 1.7* 10^308 ，存储 15/16 位有效数字
	- `long double` 长双精度浮点数，存储范围：-1.1* 10^-4932 ~ 1.1* 10^4932 ，存储 18/21、33/36 位有效数字

## 变量的声明注意
当内部作用域和外部作用域声明了同名变量时
内部作用域中的变量会屏蔽外部作用域中的变量
```c++
#include<iostream>
using namespace std;
void change();
int A = 100; //全局变量
int main(){
	cout << A; // 打印出全局变量的100
	int A = 200; // 内部声明 不会报错
	cout << A; // 打印出内部变量的200
	change(); // 调用函数 修改的是全局变量中的A
	cout << A; // 打印还是内部变量的200
}
void change(){
	A = 150;
}
```

# 内存分配

## 内存存储区域

在程序运行时，系统中会有不同的内存存储区域
它们分别是：

- 代码段（Code Segment，或称代码存储区）
	存储：函数的机器码（执行指令）等信息的区域
	通常是只读的，防止程序意外修改自己的代码
	在程序加载时就被加载到内存中
	生命周期较长，与程序同生共死

- 数据段（Data Segment，或称数据存储区）
	存储：全局变量、静态变量等信息的区域
	一般分成两个子区域
	初始化数据段：存储已初始化的全局和静态变量
	未初始化数据段（BSS Segment）：存储未初始化的全局和静态变量，通常初始化为零
	生命周期较长，与程序同生共死

- 常量段（Constant Segment，或称常量存储区）
	存储：字符串字面量和其他只读数据 信息的区域
	通常是只读的，以防止程序修改常量的值
	比如 `cout << "Hello,World" << endl` 中的 `Hello,World` 就存储在常量区
	生命周期由程序控制

- 栈（Stack）
	存储：局部变量、函数参数、返回值 等信息的区域
	由编译器自动管理，随着函数的调用和返回而分配和释放
	生命周期与函数调用相同，快速分配和释放

- 堆（Heap）
	存储 动态分配的内存数据 等信息的区域
	由程序员手动管理，需要主动分配，主动释放
	生命周期不固定，由程序员决定生死

## 内存安全

内存安全（Memory Safety）
指的是程序在访问和操作内存时不会因为出现程序问题导致程序报错、崩溃的情况

想要保证内存安全 需要保证以下几点：
- 了解内存销毁时机
	- 栈内存中
		在函数调用结束时 局部变量的内存会自动销毁。
		栈内存是程序自动管理的 一旦程序退出某个作用域（例如函数或代码块）
		该作用域内的局部变量的内存会自动释放
		注意不要在作用域外使用作用域内的在栈上分配的内存
	- 堆内存中
		使用 `new` 分配的堆内存需要在不使用后通过 `delete` 或 `delete[]` 进行释放
		注意控制堆内存对象的生命周期，不用时一定要删除
	- 栈和堆的区别
		1. 内存空间大小不同
			栈的大小通常是固定的（比如几兆） 是由操作系统决定的 因此不能在其中分配大内存
			堆的大小取决于系统可用的内存 理论上可以分配很大的内存 可以在其中分配大内存
		2. 速度不同
			栈的分配和释放速度快
			堆的分配和释放速度较慢
		3. 线程安全
			栈是线程私有的 不同线程的栈彼此独立 是线程安全的
			堆是进程共享的 多线程访问时需要保证线程安全
	- 两者应用：
		栈：适用于小规模 短生命周期的数据
		堆：使用于动态分配的大规模数据或生命周期跨越函数调用的数据

- 防止内存泄漏
	内存泄漏主要指不再被使用的内容一直占用内存资源 在使用堆内存时很容易造成这样的结果
	1. 堆内存忘记释放（一直占用堆内存）
	2. 丢失对已分配堆内存的记录（指针指向改变 导致先前申请的空间无法释放 一直占用堆内存）
	出现以上类似的情况就会造成内存泄漏
	当内存达到一定的瓶颈是就会崩溃或报错 程序无法正常运行

- 防止越界访问
	限制数组或指针的访问范围
	避免访问超出分配范围的内存

- 空指针检查
	1. 在声明指针没有初始化时对它赋值为 `nullptr`
	2. 对于指向堆内存的指针，释放后，同样对它赋值为 `nullptr`
	并且在使用指针之前 先判断它是否为 `nullptr` 空指针再决定如何使用它

- 避免使用野指针
	避免指针指向已释放的内存空间
	确定指针指向的内存不能用时 一定将该指针置空
