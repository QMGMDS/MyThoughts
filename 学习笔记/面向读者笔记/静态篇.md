
# 静态怎么来的？

都说面向对象面向对象，将万物抽象成对象，为其封装各种表示行为的成员方法和表示特征的成员变量。这意味着：如果你想要使用该类中的成员，就得先 `New` 出一个类对象
也就是说，倘若我想要将所有关于数学计算的方法封装成一个工具类方便使用
比如：
```C#
class MathTool
{
	// 取绝对值的方法
	public int Abs(int x)
	{
	    return x > 0 ? x : (-x);
	}
}
```
那么在每次使用该类内部的方法时都得：
1. 先对 `MathTool` 这个数学工具类 `new` 出一个实例
2. 才能调用类内部的方法
显然这种做法在实际开发中显得繁琐且低效

有没有什么办法能：
1. 跳过实例化对象而直接使用其内部方法
2. 保持对工具方法的封装性

先来思考一下问题怎么来的：
正是因为工具类内部方法的出生是在类产生对象后才存在的
所以无法在对象出生前就直接调用其内部方法
这是 **实例成员生命周期的问题**
对于实例成员的生命周期 = 对象的生命周期（对象出生 -> 对象销毁）
