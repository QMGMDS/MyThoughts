# 枚举怎么来的？

现在你作为程序在进行游戏开发 而你的策划告诉你他在设计一款游戏
想要在玩家奔跑状态下执行一套逻辑
而在行走状态下执行另一套逻辑
但奔跑和行走两个状态的切换关系不是线性的 而是并列的
即：要么是奔跑状态 要么是行走状态
~~总不能玩家既奔跑又行走吧（笑~~

这个时候聪明的你立马想到了解决这个问题的办法
即使用 `if else` 语句 像这样：
```C#
bool isWalk; // 玩家是否为行走状态
if(isWalk)
{
	// 玩家行走状态逻辑
}
else
{
	// 玩家奔跑状态逻辑
}
```
于是你通过修改一个表示状态的变量实现了行走和奔跑两套逻辑的切换

没问题 这样显然是正确的 你很高兴的将代码写好了并将他交给了策划
然而这个时候 策划灵机一动！
玩家不仅有行走和奔跑状态 还有受伤 死亡 甚至开技能时的霸体状态

而先前的 `isWalk` 作为布尔值显然只能实现两个状态之间的切换
毕竟布尔值只有 `true` 和 `false` 两种可能性 就像硬币只有正反两面一样
于是你发挥了想象力 很快就想到了可以使用更多可能性的变量来作为状态切换的标识
`short` `int` `long` 等等 这些变量的值有着起码上万的可能性
然后你顺手挑了个 `int` 写下了这段代码：
```C#
int playerState = 0;
if (playerState == 0)
{
    // 玩家行走状态逻辑
}
else if (playerState == 1)
{
    // 玩家奔跑状态逻辑
}
else if(playerState == 2)
{
    // 玩家受伤状态逻辑
}
```
你将行走 奔跑 受伤等状态抽象成了整数数字
0 代表行走
1 代表奔跑
2 代表受伤
这样一来完美解决了状态数量的容纳问题
而在你一直写新的 `else if` 的时候
你发现了也可以使用 `switch` 语句 就像这样：
```C#
int playerState = 0; // 作为玩家状态的切换变量
switch(playerState)
{
	case 1:
		// 玩家行走状态逻辑
		break;
	case 2：
		// 玩家奔跑状态逻辑
		break;
	case 3:
		// 玩家受伤状态逻辑
}
```
这样一来逻辑更加清晰了 你高兴的将写好的代码丢给策划
但他看不懂这里的 1 2 3 分别都表示着什么
而你每一次看 1 2 3 的时候也需要结合注释和代码内容去看这里做的什么状态处理
这样可读性太差了吧
于是你又想到了 `string` 于是乎洋洋洒洒写下：
```C#
string playerState = "行走";
switch (playerState)
{
    case "行走":
        break;
    case "奔跑":
        break;
    case "受伤":
        break;
}
```
这样一来 只要是识字的中国人都看得懂这一段代码写的时什么了吧（

然而你觉得还是不够 毕竟字符串的内容不能按 Tab 键补齐
而笨蛋的你还老是打错别字导致切换状态时出错

那有没有什么变量同时具有：
1. 足够多的可能性
2. 很好的代码阅读性
3. 可以按 Tab 自动补全的方便书写性

有的兄弟 有的！
**枚举** 应声而来~ 于是你的代码变成了：
```C#
PlayerState playerState = PlayerState.Walk;
switch (playerState)
{
    case PlayerState.Walk:
        break;
    case PlayerState.Run:
        break;
    case PlayerState.Hurt:
        break;
}
```
**枚举** 这个自定义的变量类型就同时满足了以上的三点要求
问题迎刃而解了不是吗）

但这个时候聪明的小朋友就会问了：
*我该怎么去自定义一个枚举呢？*

# 枚举的定义

- **定义的语法**
以上面的例子来说明 这样写：
```C#
enum PlayerState
{
    Walk,
    Run,
    Hurt,
}
```
可以发现 `enum` 即枚举的关键字 而 `PlayerState` 就是该枚举的名字
结构就像你定义结构体一样 区别就是里面的成员是你给状态起的名字

- **定义的位置**
需要注意的是 枚举的定义只能在
1. `namespace` 语句块中
2. `class` 语句块中
3. `struct` 语句块中
大多数情况下是将枚举定义在 `namespace` 语句块中

这样一来你就有个名为枚举的自定义的变量了~
紧接着就是......

# 枚举的使用

既然是变量 要使用的话就逃不开变量的声明
枚举变量的声明：
```C#
PlayerState playerState = PlayerState.Walk;
int i = 114514;
```
就像声明正常的变量一样
`变量类型 变量名 = 初始值`
只不过初始值的范围变成了你在定义时的 `Walk` `Run` `Hurt` 这些了

# 拓展

至此 相信你已经知道了枚举是什么 为什么 怎么用
接下来就是一些无关紧要的冷门知识拓展了

怕你懒得往上翻定义 所以在这里又写了一次
~~才不是为了水字数口牙~~

```C#
enum PlayerState
{
    Walk, // 0
    Run,  // 1
    Hurt, // 2
}
```

- 你知道吗？枚举和 `int` 的相互转换
```C#
PlayerState playerState = PlayerState.Walk;

// 枚举 转 int
// PlayerState.Walk -> 0
int i = (int)playerState;

// int 转 枚举
// 2 -> PlayerState.Hurt
playerState = (PlayerState)2;
```

- 你知道吗？枚举和 `string` 的相互转换
```C#
PlayerState playerState = PlayerState.Walk;

// 枚举 转 string
// PlayerState.Walk -> "Walk"
string str = playerState.ToString(); 

// string 转 枚举
// Run -> PlayerState.Run
playerState = (PlayerState)Enum.Parse(typeof(PlayerState), "Run");
```

聪明的你可能发现了 `string`转枚举 怎么这么复杂
你先别急 这涉及 **反射** **静态方法** **装箱和拆箱** 相关的知识点

# 总结

枚举是自定义的变量类型 通常与 `switch` 语句搭配使用形容物体的状态、类型
方便在不同分支下灵活切换 ~~偷懒的产物~~
