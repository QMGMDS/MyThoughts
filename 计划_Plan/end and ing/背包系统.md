# 1.0  实现流程


## 1.1  背包物品数据的存储

首先需要解决的便是背包内物品数据的存储
该系统将背包内物品的数据信息分为两个部分，它们分别是：

- **静态数据**
	指物品的基本属性，例如图片Icon、物品类别、描述信息等
	这里采用：
	以 *ScriptableObject* 的形式将物品静态数据存储起来

- **动态数据**
	指物品的变化属性，例如数目、物品状态等
	这里采用：
	使物品的动态数据 *JSON序列化* 后存储到 *PlayerPrefs* 中
	在加载背包内物品信息时从 *PlayerPrefs* 中拿取数据
	以此实现游戏内物品动态数据的保存


## 1.2  一套简易的UI框架

然后是搭建一套 *简易的UI框架* 实现玩家与背包之间的交互
这套框架的核心思想是：
将各个 UI界面 制作成预制体，在玩家做出某个交互时对应显示该交互所绑定的 UI界面

**UI 场景的加载与卸载：**

存在两个最基本的脚本
*BasePanel* 和 *UIManager*
前者作为所有 UI 界面的基类
内含 OpenPanel 和 ClosePanel 这两个方法，控制被挂载对象激活状态
后者作为管理所有 UI 配置信息，内含三个字典
- `pathDict` 路径配置字典
	配置所有 UI 预制体的文件路径

- `prefabDict` UI 预制体缓存字典
	配置所有加载过的 UI 预制体，在第二次打开直接从中获取

- `panelDict` 当前加载的 UI 预制体字典
	保存当前页面显示的 UI 预制体，方便处理当前界面 UI 的卸载

以及两个方法：
- UI 预制体加载 `OpenPanel`
	经过三重检查：
	- 要加载的 UI 当前是否被加载过了（在 `panelDict` 字典是否存在）
	- 要加载的 UI 输入名称是否正确或者路径是否配置
	- 要加载的 UI 是否曾被加载过
	检查完毕后才加载目标 UI 并添加到 `panelDict` 字典中

- UI 预制体卸载 `ClosePanel`
	确保要卸载的 UI 是否在打开字典中，只有被打开的 UI 才能被卸载
	卸载 UI 并清理字典

以上实现了 UI 的前期准备
接下来是配置实际的 UI 操纵

表示总背包 UI 物体上挂载的 `PackagePanel` 中
	配置好所有子物体 UI 的 Transform 组件，便于后续绑定 UI 操纵逻辑
	通过 Transform 组件找到对应 UI 子物体的 Button 组件，通过注册事件实现按钮操纵 UI


**物品实际数据的 UI 显示：**

在每一个物品的身上挂载 `PackageCell` 脚本
该脚本控制 UI 实际显示显示效果，而物品数据在 `GameManager` 中拿取

`PackageCell` 中的 `Refresh` 方法能将传入的单个物品的数据转换成 UI 显示
`GameManager` 中能拿取物品的动态数据和静态数据并内含排序方法


**整体流程是：**

背包 UI 物体 PackagePanel 在被打开时（打开背包界面时），身上的 `PackagePanel` 脚本会初始化内部子物体的 UI 组件以及绑定 Button 事件，随后在 `RefreshUI` 方法中，卸载原本背包内 `Scroll View` 的所有子物体，每个子物体代表着一个物品栏，随后通过 `GameManager` 的方法拿到物品的动态和静态数据后，遍历每个通过 `Instantiate` 方法创建出来的子物体，并调用子物体身上 `PackageCell` 脚本中内置的同步物品信息方法，传入物品的数据，实现逐个创建物品并同步信息的过程
