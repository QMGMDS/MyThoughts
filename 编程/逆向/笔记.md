
# 前言

**什么是逆向工程？**

这里直接摘录维基百科上的定义：
> 逆向工程，也称为反向工程或反向设计，是指通过解构人工制品来揭示其设计、架构、代码或从制品中提取知识的过程。它类似于科学研究，唯一的区别在于科学研究是对自然现象进行的。

简单来说，逆向工程就是一门研究已编译程序，分析其执行原理的学问
它在现代恶意软件分析、应对网络入侵的方面上有着十分重要的作用
逆向工程远不止恶意软件分析那么简单，它同样对所有的软件生效
~~有了它还可以解包游戏(~~
这里将以分析恶意软件作为主要示例，学习逆向工程的基础概念知识

像是编写程序时所用的高级语言一样，逆向工程也有自己的语言，他们都叫做汇编语言
分别在x86、x64 以及 32 位 ARM 和 64 位架构下的对应汇编语言不同
这些架构指的就是 CPU 的架构
以下是常见 CPU 架构对比表：

| 架构名称         | 位数       | 主要设计公司                      | 典型应用场景                    |
| ------------ | -------- | --------------------------- | ------------------------- |
| x86 / IA-32  | 32位      | Intel、AMD                   | 老式个人电脑、旧款服务器、某些嵌入式系统      |
| x86-64 / x64 | 64位      | Intel、AMD                   | 现代台式机、笔记本、服务器（目前的主流）      |
| ARM          | 32 / 64位 | Arm Holdings（授权给苹果、高通、联发科等） | 智能手机、平板电脑、嵌入式设备、苹果 M 系列芯片 |


那么问题来了，汇编语言是什么呢？
这就不得不提编程语言的发展体系了
- 第一代：机器语言
	纯二进制代码（例如 `10110000 01100001`），直接由CPU执行。这是计算机的母语，但人类几乎无法理解和记忆
- 第二代：汇编语言
	用简单的助记符（如 `MOV AL, 61h`）代替二进制，标志着计算机语言史上的第一次抽象。但此时，语言仍然与特定的CPU架构（如x86、ARM）紧密绑定
- 第三代：高级语言
	方便人类程序员编写而设计的编程语言。它使用接近人类语言（通常是英语）的指令和数学符号，让程序员可以专注于解决问题，而不必关心底层计算机硬件的具体细节

学习逆向工程就是学会将软件反编译得到的汇编语言看懂，而软件是在 CPU 不同架构下诞生的适配性不同的 “硬件驱动器” ，这就是为什么会有 32 位和 64 位软件，由此可见，软件是根据 CPU 架构的不同诞生的，架构不同会导致反编译对应的汇编语言不同，所以学习不同反汇编语言就要学习不同的 CPU 架构


# x86

## 不知道怎么分类零散知识

两种分析恶意软件的基本技术：
- 静态分析
	静态分析而无需实际运行反编译的汇编指令，而是直接使用软件工具来检查可执行文件

- 动态分析
	动态分析使用反汇编器和调试器在实际运行恶意软件二进制文件时进行分析

简单理解二者的区别就是：
前者不需要运行恶意程序，对反编译得到的数据进行分析能直接得到程序运行的原理
后者则需要在运行恶意程序的过程中动态的进行原理分析

这个时候就会有聪明的小朋友要问了：
那我该怎么得到反编译的数据呢？

接下来就要引出干这些活的工具了！
反编译器和调试器！
目前市场上最流行的工具叫做 IDA，它是一个跨平台、多处理器的反汇编器和调试器。市场上还有其他反汇编器/调试器工具，如 Hopper Disassembler、OllyDbg 等。

反汇编器会将用汇编、C、C++等编写的可执行二进制文件转换为可以调试和操作的汇编语言指令。

什么是x86汇编语言？
答案是它是一些列向后兼容的汇编语言
嗯...说了跟没说一样
可以得知：x86汇编语言具有很强的兼容性
而x86汇编语言有两种语法选择：
- AT&T 语法
- Intel 语法
二者的主要区别在于：
AT&T 语法中，源代码在目标代码之前
Intel 语法中，源代码在目标代码之后

内存是以字节为单位计量
每个十六进制数字有 4 位长，也称为一个 nibble
一个字节 = 8位（比特位）
一个字 = 2个字节 = 16位
一个双字 = 2个字 = 4个字节 = 32位
一个四字 = 4个字 = 8个字节 = 64位

计算机程序不过是存储在内存中的机器指令。
一个 32 位的 CPU 从内存地址中获取一个双字（双字是连续的4个字节）
它从内存中读取并加载到 CPU 中，一旦执行完毕，CPU 就会从指令指针处获取内存中的下一条机器指令


## x86 基本架构

一个计算机应用程序的本质：
是一张存储在内存中的机器指令表，构成程序的这些二进制数在 CPU 处理上结果是唯一的

x86 基本架构由 CPU、内存和 I/O 设备组成，这些设备都通过系统总线连接，如图：
![[1520249055678.jpg]]

其中，CPU 由四个部分组成：
- 控制单元
	从 CPU 中获取和解码指令，然后将它们存储和从内存中检索
- 执行单元
	指令获取和检索执行的场所
- 寄存器
	内部 CPU 内存位置，用于临时数据存储
	CPU 内的寄存器也被称之为 EPI 或指令指针，它包含从内存中获取并执行的下一个指令的地址
- 标志
	指示执行时发生的事件
![[Pasted image 20260217123702.png]]

x86 架构的 32 位 CPU 的执行过程：
32 位 CPU 会从内存的特定地址获取一个双字（4 字节或 32 位长），并将其从内存中读取并加载到 CPU 中。此时，CPU 会查看双字中的二进制位模式，并开始执行被获取的机器指令所指示的操作，指令执行完成后，CPU 会访问内存并按顺序获取下一条机器指令
整个取指和执行过程都与系统时钟相关，系统时钟是一个以精确间隔发出方波脉冲的振荡器


## 寄存器

*接下来介绍的寄存器将专注于 32 位 x86 架构*

### 通用寄存器

>通用寄存器可以用来存储任何类型的数据，其中一些寄存器获得了特定的用途，并在程序中使用。
>每一代新的通用寄存器都是为了向后兼容以前的处理器而创建的。
>这意味着在 8080 芯片上使用 8 位寄存器的代码仍然可以在今天的 64 位芯片组上运行。

共有 8 个通用寄存器：
- EAX：主要用于算术计算的寄存器。也称为累加器，因为它存储算术运算的结果和函数返回值。
- EBX：基址寄存器。指向 DS 段中的数据。用于存储程序的基址。
- ECX：计数寄存器，通常用于存储表示重复次数的值。用于循环和字符串操作。
- EDX：通用寄存器。此外还用于 I/O 操作。此外将 EAX 扩展为 64 位。
- ESI：源变址寄存器。指向由 DS 寄存器指向的段中的数据。用作字符串和数组操作中的偏移地址。它持有读取数据的起始地址。
- EDI：目标索引寄存器。指向由 ES 寄存器所指向的段中的数据（或目标）。在字符串和数组操作中用作偏移地址。它保存所有字符串操作中隐含的写入地址。
- EBP：基指针。指向栈上的数据（在 SS 段中）。它指向当前栈帧的底部。用于引用局部变量。
- ESP：栈指针（位于 SS 段）。它指向当前栈帧的顶部，用于引用局部变量。


### 段寄存器

> 段寄存器专门用于引用内存位置，系统内存的访问有三种不同方法。

共有 6 个段寄存器：
- CS：代码段寄存器存储代码段（.text 段）的基位置，用于数据访问。
- DS：数据段寄存器存储变量的默认位置（.data 段），用于数据访问。
- ES：用于字符串操作的附加段寄存器。
- SS：栈段寄存器存储栈段的基位置，在隐式使用栈指针或显式使用基指针时使用。
- FS：附加段寄存器。
- GS：附加段寄存器。

### 指令指针寄存器

>被称为 EIP 寄存器的指令指针寄存器负责跟踪下一个要执行的指令代码。
>如果你修改这个指针以跳转到代码中的其他区域，你将完全控制该程序。
