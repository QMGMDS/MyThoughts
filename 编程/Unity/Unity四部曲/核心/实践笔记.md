# 数据持久化 JSON

## JSON 基本语法

符号相关对应的含义：

| 符号 | 含义           |
| :--- | :------------- |
| `{}` | 对象           |
| `[]` | 数组           |
| `:`  | 键值对对应关系 |
| `,`  | 数据分割       |
| `""` | 键名/字符串    |

假如有一个定义好的类 Test

```C#
public class Test
{
    public string name;
    public int age;
    public bool sex;
    public List<int> ids;
    public List<Person> students;
    public Home home;
    public Person son;
}

public class Person
{
	public string name;
	public int age;
	public bool sex;
}

public class Home
{
    public string address;
    public string street;
}
```

下面是对 Test 类对象的 Json 文件表示

```json
{
    "name": "曹哲",
    "age": 18,
    "sex": true,
    "testF": 1.4,
    "ids": [1, 2, 3, 4],
    "students": [
        { "name": "小红", "age": 10, "sex": false },
        { "name": "小明", "age": 8, "sex": true },
        { "name": "小强", "age": 7, "sex": true }
    ],
    "home": { "address": "成都", "street": "超人" },
    "son": null
}
```

## 根据文件路径存读字符串

```C#
// 1.存储字符串到指定路径文件中
// 参数一：存储的路径
// 参数二：存储的字符串内容
// 注意：第一个参数 必须是存在的文件路径 如果没有对应文件夹会报错
File.WriteAllText(Application.persistentDataPath + "/Test.json", "唐老狮存储的json文件");

// 2.在指定路径文件中读取字符串
string str = File.ReadAllText(Application.persistentDataPath + "/Test.json");
```

## JsonUtlity

- `JsonUtlity` 是什么？
  `JsonUtlity` 是 Unity 自带的用于解析 Json 的公共类
  使用该类里面的方法
  可以将对象序列化为 Json 格式的字符串
  或是将 Json 格式的字符串反序列化为类对象

序列化：

```C#
MyData d = new MyData();

// 把类对象序列化为 Json 字符串
string jsonStr = JsonUtility.ToJson(d);
// 将 Json 字符串存储到硬盘中
// Application.persistentDataPath 目录下会显示名为 MyData 的 Json 文件
File.WriteAllText(Application.persistentDataPath + "/MyData.json", jsonStr);
```

注意：

1. float 序列化时会有一些误差
2. 自定义类序列化时需要加上序列化特性 \[System.Serializable]
3. 想要序列化私有变量，需要加上特性 \[SerializeField]
4. JsonUtility 不支持序列化字典
5. JsonUtlity 存储 null 对象不会是 null 而是默认值的数据

反序列化：

```C#
// 读取该路径中的 Json 文件
jsonStr = File.ReadAllText(Application.persistentDataPath + "/MyData.json");
// 将 Json 字符串内容转换成类对象
MyData d = JsonUtility.FromJson(jsonStr, typeof(MyData)) as MyData;
MyData d = JsonUtility.FromJson<MyData>(jsonStr);
```

注意：

1. 如果 Json 中数据少了，读取到内存中类对象中时不会报错，而是默认值
2. 无法直接读取数据的集合，但可以通过对象包裹的方式对之读取
3. 读取文件的编码格式必须得是 UTF-8 才行，否则报错

## LitJson

- `LitJson` 是什么？
  `LitJson` 是用C#编写的一个第三方库，用于处理 Json 的序列化和反序列化，体积小、速度快、易于使用
  只需要将 LitJson 代码拷贝到 Unity 工程中即可开始使用里面提供的方法

序列化：

```C#
MyData d = new MyData();

// 把类对象序列化为 Json 字符串
string jsonStr = JsonMapper.ToJson(d);
// 将 Json 字符串存储到硬盘中
// Application.persistentDataPath 目录下会显示名为 MyData 的 Json 文件
File.WriteAllText(Application.persistentDataPath + "/MyData.json", jsonStr);
```

注意：

1. 相对 JsonUtlity 不需要加特性就可以序列化自定义类
2. 不能序列化私有变量
3. 支持字典类型，字典的键建议都是字符串，因为 Json的特点，Json中的键会加上双引号
4. 需要引用 LitJson 命名空间
5. LitJson 可以准确的保存 null 类型

反序列化

```C#
jsonStr = File.ReadAllText(Application.persistentDataPath + "/MyData.json");
// JsonData 是 LitJson提供的类对象 可以用键值对的形式去访问其中的内容
JsonData d = JsonMapper.ToObject(jsonStr);
print(data["name"]);
print(data["age"]);
// 或者通过更方便的泛型来读取 Json 数据
MyData d = JsonMapper.ToObject<MyData>(jsonStr);
```

注意：

1. 类结构需要无参构造函数，否则反序列化时报错
2. 字典虽然支持 但是键在使用为数值时会有问题 需要使用字符串类型
3. LitJson 可以直接读取数据集合
4. 读取文件的编码格式必须得是 UTF-8 才行，否则报错
