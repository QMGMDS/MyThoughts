# 项目概要介绍

## 前言
OfferFlow 是一款面向求职训练场景的 Android 智能模拟面试应用，目标是把“练习-反馈-复盘-提升”做成可持续的闭环。项目围绕真实面试流程设计：候选人选择岗位后发起会话，系统以 AI 面试官进行多轮追问，结束后自动生成结构化评估报告，并沉淀为长期成长数据。

## 创意描述
本项目的创意核心是“流程化 AI 面试教练”。区别于单轮问答工具，OfferFlow 将面试拆分为开场介绍、正式问答、候选人反问、报告生成四个阶段，并通过流式事件驱动状态切换。创新点在于：
1. 将 AI 对话与面试业务节点绑定，形成可控、可解释的训练路径。
2. 结合语音识别与文本输入，降低练习门槛，提高真实感。
3. 报告从自由文本升级为结构化数据，直接支持统计分析与连续复盘。

## 功能简介
1. 智能模拟面试：支持岗位选择、连续追问、实时流式回复与语音播放。
2. 语音能力：支持录音、语音转文字、面试官语音播报，兼顾效率与沉浸感。
3. 报告生成：面试结束后自动触发评估，输出总分、维度评价、核心问题、改进路线等内容。
4. 数据沉淀：本地保存会话、报告、对话记录与会话上下文，支持中断后继续练习。
5. 统计复盘：提供完成率、平均分、趋势、分布、岗位构成、近期报告等多维可视化。

## 特色综述
OfferFlow 兼具创新性与实用性。创新性体现在“节点化工作流+流式交互+结构化评估”的组合；实用性体现在移动端可随时训练、支持文本与语音双通道、对练习结果可追踪。项目完整性较高，已覆盖首页、会话页、报告页、记录页、统计页和个人页。规范性与专业性体现在分层架构（UI/ViewModel/Repository/Local DB）、状态机式意图驱动、异常兜底与单元测试覆盖。逻辑链路清晰：输入行为→AI面试过程→报告产出→统计复盘→下一轮优化。

## 开发工具与技术
1. 平台与语言：Android、Kotlin、Jetpack Compose。
2. 架构与数据：MVVM、Room 持久化、Flow/协程异步流。
3. 网络与集成：Retrofit + OkHttp，接入 Dify 流式对话服务。
4. 语音能力：百度语音识别（ASR）+ 应用内音频播放。
5. 文本与可视化：Markdown 解析渲染、Canvas 统计图表。
6. 质量保障：针对评分解析、统计聚合、会话状态流转建立单元测试。

## 应用对象
1. 求职者：用于校招、社招、转岗与晋升前的面试准备。
2. 高校与培训机构：用于课程实训、面试演练与学习成果评估。
3. 企业招聘与人才发展团队：用于面试前预训练、候选人辅导与能力成长跟踪。

## 应用环境
1. 终端环境：Android 7.0 及以上移动设备。
2. 运行条件：具备麦克风权限与网络连接。
3. 配置要求：需在本地配置 Dify API Key 与百度语音密钥。

## 结语
OfferFlow 不仅是“能对话”的模拟面试工具，更是“能闭环提升”的训练系统。项目在技术上具备可实现、可扩展、可运维特征，在业务上具备用户持续使用价值与服务变现潜力，可进一步扩展为面向个人与机构的智能求职训练平台。
