# PlayerPrefs 数据持久化

- 引入：
	PlayerPrefs 作为 Unity 提供的类
	里面含有数据保存 数据加载 数据删除的静态方法
	PlayerPrefs 会将数据存储到本地注册表中以键值对的形式存在
	这种存储数据的方法适用于：
	保存一些不重要的数据
	保存一些较小的数据
	
	- 缺点：
	1. 安全性低 数据容易被串改
	2. 只提供了存储 int float string 三种类型数据的方法

- 项目概述：
	根据反射的原理
	利用 PlayerPrefs 实现对自定义类的存储
	不管类内包含
	简单数据如：int float string bool 
	复杂数据如：自定义 class 泛型类 list 和 dictionary

- 最终效果为：
	只需要调用 `SaveData(object data, string keyName)` 方法
	自行拟定 `keyName` 它将成为读取数据的唯一标识
	便可以将想要持久化的数据存储在 **Windows本地注册表** 中
	
	只需要调用 `LoadData(Type type, string keyName)` 的方法
	传入先前拟定的 `keyName` 
	便可以返回用 `object` 父类所装载的对应数据 实现本地数据读取

- 详细见：
	[[学习笔记/编程/Unity/Unity四部曲/入门/实践笔记#数据持久化 PlayerPrefs]]

- 成果：
	![[PlayerPrefsDataMgr.unitypackage]]

- 注意：
	该方法只适用保存于自定义的类
	如果要直接存储 `int` `float` `string` `bool` 等数据无法使用该方法 需要自行重新书写代码
	并且在该类中目前只支持
	`int` `float` `string` `bool` 这些基本数据类型
	`List` `Dictionary` 这些泛型类 支持自定义类型例如 `List<Itme>`
	`class` 嵌套类
	
	含数组、枚举的自定义类不能用该方法进行数据保存

- 拓展思路：
	- 添加数组数据持久化
	- 添加枚举数据持久化

# GUI

- 引入：
	GUI 即为代码驱动的 UI 系统
	Unity 提供了 GUI 这个类 里面包含很多绘制 UI控件 的静态方法
	学习 GUI 就是去学习如何使用这些静态方法
	由于它的特点和使用不方便
	导致它一般作为程序员的调试工具 而不是玩家 UI 显示
	可以为脚本组件创建自定义检视面板
	或创建新的编辑器窗口和工具以拓展 Unity 本身
	
	- 缺点：
		1. GUI 本身需要游戏运行时才能看到 UI 显示效果
		2. GUI 提供的控件本身不能很好的控制相互之间的层级关系
		3. GUI 本身无法实现分辨率自适应的效果

- 项目概述：
	解决 GUI 的缺点指出的问题
	但由于 GUI 本身不方便使用的局限
	该项目只是为了解高级 UI 底层运作原理 所以实用性不大
	- 根据九宫格原则 屏幕九宫格和控件中心位置九宫格创建出控件位置信息类 专门计算相对于当前坐标系中心偏移 解决了控件位置坐标分辨率自适应的问题
	- 创建所有基本控件的抽象父类 提高了代码的复用率
	- 创建 `CustomGUIRoot` 使得被挂载对象成为高级UI中画布一样的效果 让子对象的控件绘制统一到一个周期函数 `OnGUI()` 中 解决了 GUI 绘制层级关系无法自由控制的问题 并添加 `[ExecuteAlways]` 特性使得绘制在编辑模式下也能运行 使得 GUI 也能所见即所得
	再创建继承抽象父类的基本控件 文本 按钮 多选框 输入框 拖动条 图片 这些控件
	将其封装成预制体 至此 GUI 也能像高级 UI 一样灵活使用

- 最终效果为：
	一套基于 GUI 的可视化 UI 封装 支持编辑模式运行 提升调试效率
	一定程度上解决了 GUI 自身的缺点
	让 GUI 能像高级 UI 一样方便可视的创建调整

	作为一次底层原理实践
	GUI 一般不作为玩家 UI

- 拓展思路：
	该项目只实现了控件位置的分辨率自适应 没有实现控件大小的分辨率自适应
