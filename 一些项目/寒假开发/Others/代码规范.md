# 代码规范

- 命名清晰：使用描述性强的命名，让代码自我解释。
- 简洁性：力求简洁，避免冗余，用最少的代码行数完成功能。
- 一致性：保持项目中命名和编码风格的统一，减少认知负荷。
- 注释：用注释阐明代码意图，但避免过度注释。
- 避免复杂性：将复杂逻辑分解为简单、可管理的函数或模块。
- 重构：定期重构，提升代码的可读性和性能。
- 测试：编写单元测试，确保代码的稳定性和可靠性。
- 错误处理：合理处理错误，增强程序的健壮性。
- 文档：编写清晰的文档，包括 API 文档和项目文档。
- 代码复用：创建可复用的函数或模块，避免重复代码。
- 性能优化：在不牺牲可读性的前提下，优化性能瓶颈。
- 安全性：编写安全的代码，防范常见的安全漏洞。

接下来我们展开说明：

### 1、命名清晰

在编程中，命名是第一印象。

好的命名可以让人一眼看出变量、函数或类的作用。

清晰准确的命名可以减少误解，提高代码的可读性。

比如，`playerHealth` 比 `hp` 更清楚地表达了它存储的是玩家生命值。

变量、函数和类的命名应该直观地描述其功能和用途，避免使用模糊或不相关的名称。

### 实例

```csharp
// 好的例子：命名清晰
public float CalculatePlayerSpeed(float baseSpeed, float multiplier)
{
    return baseSpeed * multiplier;
}

// 坏的例子：命名不清晰
public float Calc(float a, float b)
{
    return a * b;
}
```

### 2、简洁性

简洁的代码意味着用最少的代码行数完成所需的功能，它减少了维护的难度和出错的概率。

例如，使用 Unity 的 `Vector3.Distance` 可以比手动计算两点距离更简洁。

尽量用最少的代码完成功能，避免冗余，简洁的代码更易于阅读和维护。

### 实例

```csharp
// 好的例子：使用内置方法
float distance = Vector3.Distance(player.position, enemy.position);

// 坏的例子：冗余计算
float dx = player.position.x - enemy.position.x;
float dy = player.position.y - enemy.position.y;
float dz = player.position.z - enemy.position.z;
float distance = Mathf.Sqrt(dx * dx + dy * dy + dz * dz);
```

### 3、一致性

一致性是团队协作中的关键。

无论是命名规则、函数结构还是代码格式，一致的风格可以减少团队成员之间的沟通成本。

我们团队使用驼峰命名法，所有的变量和函数名都应遵循这一规则。例如类名使用大驼峰命名，而变量使用小驼峰命名。

整个项目中应保持一致的命名和编码风格，包括命名约定、代码格式和注释风格。

### 实例

```csharp
// 好的例子：一致的命名
public string GetPlayerName(Player player)
{
    return player.Name;
}

public int GetPlayerScore(Player player)
{
    return player.Score;
}

// 坏的例子：不一致的命名
public string getName(Player player)
{
    return player.Name;
}

public int GetScore(Player player)
{
    return player.Score;
}
```

### 4、注释

注释是代码的说明书。

合理的注释可以解释代码的意图，帮助他人（或未来的你）理解复杂的逻辑。

最好的代码是自解释的，只有当代码本身不足以清晰表达时，才需要注释。

避免过度注释显而易见的代码。

### 实例

```csharp
// 好的例子：必要的注释
// 检测玩家是否处于无敌状态
if (player.IsInvincible)
{
    // 无敌状态下不受伤害
    return;
}

// 坏的例子：过度注释
public int Add(int a, int b)
{
    // a 是第一个数
    // b 是第二个数
    // 这个函数返回两个数的和
    return a + b;
}
```

### 5、避免复杂性

复杂的代码难以理解和维护。

尽量将复杂逻辑分解成简单的部分，使用函数或类来封装。

避免过长的函数和深层的嵌套，它们会增加代码的阅读难度。

将复杂逻辑分解为更小的、可管理的部分。

### 实例

```csharp
// 好的例子：简单的逻辑
public bool IsPlayerAlive(int health)
{
    return health > 0;
}

// 坏的例子：复杂的逻辑
public bool CheckPlayerStatus(int health, bool isInvincible, bool isHidden)
{
    if (health <= 0)
    {
        return false;
    }
    else if (isInvincible)
    {
        return true;
    }
    else if (isHidden)
    {
        return false;
    }
    else
    {
        return true;
    }
}
```

### 6、重构

重构是改进现有代码而不改变其外部行为的过程。

定期的重构可以提高代码的可读性和性能，去除重复代码，优化结构。

重构需要谨慎进行，确保测试覆盖以避免引入新的错误。

我们这次的团队协作应该不需要进行重构。

### 实例

```csharp
// 重构前：重复的字符串拼接
public string GetPlayerGreeting(string name)
{
    return "Welcome, " + name + "!";
}

public string GetPlayerVictory(string name)
{
    return "Congratulations, " + name + "!";
}

// 重构后：使用字符串插值
public string GetPlayerGreeting(string name)
{
    return $"Welcome, {name}!";
}

public string GetPlayerVictory(string name)
{
    return $"Congratulations, {name}!";
}
```

### 7、测试

单元测试是确保代码按预期工作的保障。

编写单元测试，确保代码的稳定性和可靠性。

测试可以自动验证代码的功能，特别是在代码修改或重构时。

测试可以根据自己的情况进行。如果某个脚本是专门用于测试功能的，请标注。

### 实例

```csharp
// 使用NUnit框架编写测试
using NUnit.Framework;

[TestFixture]
public class PlayerTests
{
    [Test]
    public void TakeDamage_ReducesHealth()
    {
        Player player = new Player(health: 100);
        player.TakeDamage(20);
        Assert.AreEqual(80, player.Health);
    }

    [Test]
    public void Heal_IncreasesHealth()
    {
        Player player = new Player(health: 50);
        player.Heal(30);
        Assert.AreEqual(80, player.Health);
    }
}
```

### 8、错误处理

错误处理是健壮程序的重要组成部分。

恰当地处理可能发生的错误情况，避免程序在遇到异常情况时崩溃，同时提供有用的反馈。

常见的例如判空、空引用、目标销毁等。

### 实例

```csharp
// 好的例子：恰当的错误处理
public void LoadPlayerData(string path)
{
    try
    {
        string data = File.ReadAllText(path);
        // 处理数据
    }
    catch (FileNotFoundException ex)
    {
        Debug.LogError($"存档文件未找到: {ex.Message}");
    }
    catch (Exception ex)
    {
        Debug.LogError($"读取存档失败: {ex.Message}");
    }
}

// 坏的例子：缺乏错误处理
public void LoadPlayerData(string path)
{
    string data = File.ReadAllText(path); // 没有错误处理
}
```

### 9、文档

文档是项目的地图。

编写清晰的文档，包括 API 文档和项目文档，帮助新团队成员快速了解项目结构，API 文档则让使用者了解如何使用你的代码。

写好了的方法最好都写一个API。

### 实例

```csharp
/// <summary>
/// 玩家控制器类，负责处理玩家输入和移动逻辑。
/// </summary>
public class PlayerController : MonoBehaviour
{
    /// <summary>
    /// 移动玩家到指定位置。
    /// </summary>
    /// <param name="targetPosition">目标位置（世界坐标）。</param>
    /// <param name="speed">移动速度（单位/秒）。</param>
    /// <returns>是否成功开始移动。</returns>
    public bool MoveTo(Vector3 targetPosition, float speed)
    {
        // 实现移动逻辑
    }
}
```

文档中清晰地说明了类的作用、参数和返回值，帮助其他开发者正确使用这个类。

### 10、代码复用

避免重复是编程的一条基本原则。

避免重复编写相同的代码，创建可复用的函数或模块，减少代码的冗余，提高开发效率。同时，复用的代码更容易维护和更新。

### 实例

```csharp
// 好的例子：复用代码
public interface IDamageable
{
    void TakeDamage(int amount);
}

public class Player : MonoBehaviour, IDamageable
{
    public void TakeDamage(int amount)
    {
        // 玩家受伤逻辑
    }
}

public class Enemy : MonoBehaviour, IDamageable
{
    public void TakeDamage(int amount)
    {
        // 敌人受伤逻辑
    }
}

// 坏的例子：重复代码
public class Player : MonoBehaviour
{
    public void TakePlayerDamage(int amount)
    {
        // 玩家受伤逻辑
    }
}

public class Enemy : MonoBehaviour
{
    public void TakeEnemyDamage(int amount)
    {
        // 敌人受伤逻辑
    }
}
```

### 11、性能优化

性能优化是提高程序运行效率的过程。

在不牺牲可读性的前提下，对性能瓶颈进行优化。这可能涉及到算法的选择、数据结构的使用或者代码的优化。

但记住，过早的优化是万恶之源，确保在不牺牲代码可读性的前提下进行优化。

我们的项目在不影响游戏性能的情况下都不考虑优化。

### 实例

```csharp
// 好的例子：使用对象池管理子弹
public class BulletPool : MonoBehaviour
{
    private Queue<GameObject> bulletPool = new Queue<GameObject>();

    public GameObject GetBullet()
    {
        if (bulletPool.Count > 0)
        {
            return bulletPool.Dequeue();
        }
        return Instantiate(bulletPrefab);
    }

    public void ReturnBullet(GameObject bullet)
    {
        bulletPool.Enqueue(bullet);
    }
}

// 坏的例子：频繁实例化与销毁
public class Shooter : MonoBehaviour
{
    public void Shoot()
    {
        GameObject bullet = Instantiate(bulletPrefab);
        // 使用子弹
        Destroy(bullet, 5f);
    }
}
```

### 12、安全性

安全性是编程中不可忽视的方面。

编写安全的代码，避免常见的安全漏洞，如 SQL 注入、XSS 攻击等。

不知道是什么就不管了。

### 实例

```csharp
// 好的例子：使用参数化查询防止SQL注入
public PlayerData LoadPlayerData(string username)
{
    string query = "SELECT * FROM Players WHERE Username = @Username";
    using (SqlCommand cmd = new SqlCommand(query, connection))
    {
        cmd.Parameters.AddWithValue("@Username", username);
        // 执行查询
    }
}

// 坏的例子：易受SQL注入攻击
public PlayerData LoadPlayerData(string username)
{
    string query = $"SELECT * FROM Players WHERE Username = '{username}'";
    // 执行查询
}
```
